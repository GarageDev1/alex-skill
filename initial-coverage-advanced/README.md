# initial-coverage-advanced

生成机构级中文**首次覆盖（深度）研报及配套 Excel 模型**的 Claude Code skill。对标中泰、华创、东吴、银河等券商深度报告，最终交付联动 Excel + DOCX：Excel 承载盈利预测、估值、同业及产业链横向比较的完整数据与公式；DOCX 承载面向读者的分析和最终投资结论。报告体量为 25–45 页、1.5–3 万字、40–70 个图表，覆盖 A 股、港股、美股，按标的所在市场自适应会计准则、币种和估值惯例。

迭代的是研报生成的**方法论**（`SKILL.md` + `references/` + `assets/`），不是某一份具体报告。

## 仓库结构

| 文件 / 目录 | 作用 |
|---|---|
| `SKILL.md` | skill 入口：触发条件、五阶段总览、取数与输出标准 |
| `references/` | 每个阶段的详细做法，外加 Word 格式标准、估值方法学及同业与产业链横向比较规范 |
| `assets/` | 报告模板与版式、研报写作规范、去AI味案例库、质量检查清单 |
| `scripts/anti_fabrication_lint.py` | 交付前反编造检查脚本 |

## 五阶段流程

| 阶段 | 名称 | 前置 | 产物 |
|---|---|---|---|
| 1 | 公司研究 | 公司名 / 代码 | 公司研究稿 `.md` |
| 2 | 盈利预测建模 | 能获取财报 | 财务模型 `.xlsx` |
| 3 | 估值与目标价 | 阶段 2 模型 | 估值稿 `.md` + 估值页及 O/V/T 横向比较 sheets 并入 Excel |
| 4 | 图表生成 | 阶段 1、2、3 | 图表包 `.zip`（PNG + 清单） |
| 5 | 报告组装 | 阶段 1–4 全部 | 研报 `.docx`；全流程交付同时保留联动 Excel 模型 |

阶段 3、4、5 依赖前序产物，开始前先校验前置；缺了就停下补，不拼占位内容。

## 直接调用时怎么走

用户直接说“给某公司做一份首次覆盖”时，skill 按以下链路连续推进；每个阶段产物落地并自检后再进入下一阶段，只有前置或关键数据缺失时才停下：

```text
阶段 1：公司研究 + 建立 O/V/T 候选池
→ 横向数据先增量写入 staging 并校验
→ 阶段 2：建立分部盈利预测 Excel
→ 阶段 3：把估值与 O/V/T 横向比较合并进同一个 Excel
→ 阶段 4：只从已验证 Excel 快照生成图表
→ 阶段 5：Word 读取 Excel 的关键数据与图表，写出最终投资结论
```

## 修改前后对比

| 维度 | 修改前 | 修改后 |
|---|---|---|
| 可比公司定义 | 笼统使用 5–10 家可比公司 | 分为 O 经营对标、V 严格估值可比、T 产业链主题三层 |
| 横向数据落点 | 分散在研究稿、估值页、图表和 Word | 完整数据、公式、来源和检查统一落在 Excel |
| 数据更新 | 可能直接更新正式表格 | 先写独立 staging，校验后增量合并，不覆盖源数据 |
| 估值样本 | 同行、直接竞品和主题股可能混用 | 只有 V 池可以进入估值分位、目标倍数和 SOTP 分部倍数 |
| 产业链比较 | 以产业链图或文字说明为主 | T 池横向比较市值、收入、利润、PE、PS、增长和主题暴露 |
| 图表数据源 | 可能从研究稿或外部数据重新取数 | 阶段 4 只能读取同一个已验证 Excel 快照 |
| Word 的职责 | 汇总文字、模型数字和图表 | 从 Excel 摘取必要证据，并强制形成 O/V/T 最终投资结论 |
| 最终交付 | 主要是一份 DOCX | 联动 Excel 模型 + DOCX 研报 |

## O/V/T 横向比较：Excel 与 Word 如何分工

### Excel：正式数据产物与唯一数字底座

横向比较必须并入阶段 2/3 的同一个盈利预测与估值工作簿，至少包含四个 sheets：

| Sheet | 内容 |
|---|---|
| `横向比较_主数据` | O/V/T 候选全集、唯一证券 ID、行情与财务原值、币种/期间、来源、纳入理由、公式和检查列 |
| `O_经营对标` | 直接竞品的份额、产品、客户、产能、增长、利润率和关键经营指标 |
| `V_估值可比` | 严格可比公司的市值、收入、利润、PE、PS、EV 倍数、分位、目标倍数和溢折价 |
| `T_产业链主题` | 核心概念股及产业链公司的环节、关系、规模、增长、利润、PE/PS 和主题暴露 |

后三个视图必须由主数据通过 Excel 表、公式、Power Query 或明确的跨表引用派生，不能手抄第二套数字。只有 V 池公司可以进入目标倍数；O/T 公司只有同时独立满足 V 池标准时才能参与估值。

例如研究 SK hynix：Micron 可作为 O/V 候选；Samsung 存储业务属于 O，Samsung Electronics 合并口径是否进入 V 要先处理集团业务污染；NVIDIA、TSMC 属于 T，不能因为同属 AI 产业链就直接用于 SK hynix 的目标 PE。

### Word：最终投资结论载体

Word 不保存另一套完整横向数据，但必须从 Excel 摘取必要表格、图表和数字证据，并写入三组结论：

1. **O 池结论**：相对直接竞品的经营强弱、优势形成机制、短板及对盈利预测的影响。
2. **V 池结论**：当前估值的溢价/折价、是否合理、目标倍数依据及估值组迁移变量。
3. **T 池结论**：公司所在产业链利润池、跨环节定价差异及关键变量向盈利和估值的传导。

每组结论至少有一条数字或图表证据，并落到盈利预测、目标价、评级或风险。Excel 四个 sheets 不齐、检查未通过，或 O/V/T 任一组结论没有进入 Word，均不得完成最终交付。

## 本版执行要求

**数据不许编造。** 取不到的数留空或标「未获取」，不合成、不插值、不用"示意"填一个看着合理的值。正文数字和图表数据点用同一把尺，不分承重次要。每个数先进 `_data` 底稿（数值、口径、时间点、来源、置信度），底稿里没有的不落笔。交付前跑 `scripts/anti_fabrication_lint.py`，命中逐条清掉才算过。这道门和"渲染成 PDF 视觉核验"同级。

**股价数据回结算一手源。** 现价、指数、市值取交易所 / 行情接口的结算日线收盘，区分盘中价和收盘价。画股价图一律用日频逐日真实收盘，不周频重采样、不插值。

**写作去 AI 味。** 正文按 `assets/研报写作规范.md` 写，避开套话、伪分析、三段排比、硬翻译、情绪化比喻；拿不准对照 `assets/去AI味案例库.md` 的真实改稿。

**横向比较不能混池。** 经营竞品不自动等于估值可比，同一 AI/新能源主题也不等于可以共用目标倍数。完整数据和公式留在 Excel，最终结论必须进入 Word。

## 本地安装

clone 到 Claude Code 的 skills 目录：

```bash
git clone https://github.com/angelo-xuang/initial-coverage-advanced.git ~/.claude/skills/initial-coverage-advanced
```

之后在 Claude Code 里说"给某公司写一份首次覆盖 / 深度研报"就能触发。只要某一个阶段也行，直接说要哪一阶段，它会先校验前置再做。

## 协作迭代

改方法论时：

1. 改对应的 `references/阶段N-*.md`，或 `assets/` 里的规范、模板。
2. 用一个真实标的端到端跑一遍，确认产物质量。
3. commit message 写清改了哪个阶段、解决什么问题。
