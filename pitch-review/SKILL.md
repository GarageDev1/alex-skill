# pitch-review — 推票周度复盘

复盘的目的是**验证判断、沉淀方法论**：上周按持仓期推的票，本周兑现得怎么样，推票逻辑站不站得住。按下面工作流逐步做，每步的规则写在该步里。

---

## Step 0 · 定位输入与口径

1. **拿到上周推票清单**：一份含短期/中期/长期三档标的的推票笔记（或用户直接给）。抽出每只的：**推票价、推票立场（推荐/警示/等回调/买点区间）、持仓期分档**。这是复盘的对照基准，本 skill 不改推票原文。
2. **确定复盘区间**：用户给（如 6/29–7/8）或默认"上次推票日至今"。**区间涨跌基准 = 区间前最后一个收盘**（6/29 起则基准 6/26 收盘）。
3. **确定输出文件**：用户指定的 md；若已有大纲，默认只填 1–4 板块，不动第五节（下周推荐）、第六节（风险总结）。**要求写第五节时走 Step 5B**。
4. 文首写一行**数据口径**：区间基准、各市场截止日（美股常比亚洲少一个交易日，写明）、行情源（如"K线源 Yahoo，前复权"）。
5. **改报告前先备份**：写入前把目标 md 拷一份到临时目录，改坏可回滚；大段/多图替换用脚本按锚点定位、别手改（长文手改易错）。

## Step 1 · 取真实数据（当次取、别用记忆里的旧数）

全部走 `scripts/fetch_bars.py`（Yahoo，免 key），取数配方见 `references/data-and-charts.md`：
- **指数 + 参考资产 + 个股 + 行业 ETF**：`python scripts/fetch_bars.py --interval 1d --range 1mo --spec <代码...> > spec.json`（拿 bars，既算区间涨跌又画日K）。
- **A 股申万一级 31 行业全景**：Yahoo 无申万行业指数——用免费源（如 akshare）或任何满足契约的结构化源补，与美股 GICS **同口径同时窗**。
- **板块催化 / 领涨领跌股 / 事件**：用宿主的 Web 搜索，给数字配机制与催化。
> 规则：关键数字双源或回一手核对；任何密钥只在临时命令的环境变量里用，绝不写进报告或本 skill。

## Step 2 · 写 Part 1 开篇导语（定调）

- 一句话核心判断（本期市场是什么性质的行情，主线是什么）。**若是轮动/切换行情，结论必须是资金"多方向扩散"的真实图景，不能简化成"从 X 切到 Y"**（去向要落到 Step 1 的全市场数据）。
- 2–4 条主线 bullet（每条带关键数字 + 催化）。
- **对上周推票的意义**：一句话预告三档推票本周兑现如何（承接第四节）。
- 本期结构预告。

## Step 3 · 写 Part 2 大盘复盘（定基调）

- **跨市场区间涨跌表/图**（模板①）：A股（上证/深成/创业板/沪深300/科创50）、港股（恒生/恒生科技/国企）、美股（道指/标普/纳指/费半 SOX）。
- **分市场散文**：每个市场讲区间涨跌 + 走势特征 + 驱动；**关键指数就近内联日K（模板③，高亮复盘区间）**——讲 A 股放上证/科创50 的日K，讲港股放恒生科技，讲美股放费半 + 标普。
- **参考资产表**：VIX / 美债 10Y / 美元 / 黄金 / 原油——用来解释资金情绪与风格切换（如 VIX 低=结构轮动非系统性恐慌）。
- **小结**：资金主线一句话（多方向扩散时列出各去向），风险偏好判断。

## Step 4 · 写 Part 3 板块复盘（找方向）

- **两张对称全景 bar（模板④）**：美股 GICS 11 行业 + A 股申万一级 31 行业，同口径同时窗。这是"多方向扩散"的硬证据。
- **领跌端**：通常集中在少数拥挤的高估值方向（点名，别笼统说"科技"——分清"半导体"与"科技"）。
- **领涨端**：分散、多点开花，每个方向标**独立催化**（如创新药=医保目录、黄金=避险、红利=资源/反内卷）。
- **一级 vs 二级/个股**：主题强势常在二级/个股，一级口径可能相反（如贵金属个股大涨但申万有色一级仍跌）——讲清楚，别被个股误导成板块信号。
- 收一句：这轮卖压是否砸在上周推票覆盖的板块上，自然引出第四节。

## Step 5 · 写 Part 4 个股复盘（验证判断 —— 用推票口径）

- **总览**：区间跌幅 bar（模板⑤，按跌幅排序）+ rebased 折线（模板②）。
- **按短/中/长期分档**逐只，每只四段式：**区间股价变化 → 催化事件 → 推票逻辑是否兑现 → 结论**。
  - **必须用推票本身的逻辑**：它当时推荐/警示了什么、买点区间、等回调、本股vs ADR 这类操作建议。**不要**搬深度报告内部逻辑（风险编号、PE 阶梯、概率加权情景、护城河长篇）。
  - 每只就近配**高亮日K（模板③）**，把推票价所在位置和复盘区间对上。
  - 诚实记录：兑现的说兑现，偏差的直接说偏差（复盘的价值就在认错），但用干净陈述、不写"需诚实标注"这类元话语。
- **复盘小结**：一句话按持仓期/估值定位给出跌幅排序与规律（如"越短期越透支跌越狠、越长期越合理越抗跌"）；列"对的/偏的/对下周的铺垫"。

## Step 5B · （可选）Part 5 下周个股推荐

仅当报告要求写"下周推荐"时做——把**已定的标的清单**（用户给的 ticker + 持仓期分档，**跨市场：美/港/A，不限 A 股**）写成图文 + 框架齐全的推荐段。本 skill 不做全市场选股，只把清单写实。逐只：

1. **标的核验（先做，别拿记忆里的 ticker 认知当现状）**：每个代码先核实当下真身——是否退市/被收购/借壳改名（例：`CCXI` 已非老 ChemoCentryx，现为 Churchill Capital XI 借壳 Agility Robotics）、是否近期拆股（现价与印象差一个量级就查）、是否 SPAC/pre-close de-SPAC（用事件驱动框架，不套经营公司模板）。用 Web 搜索查最新事件，`fetch_bars.py` 拉现价对齐。
2. **逐票深研（并行子 agent）**：每家派一个 web-only 子 agent，强制当次最新、带出处，产出结构化八段：业务本质 / 最新催化（带日期）/ 估值锚（现价·PE TTM+fwd·卖方一致目标·评级分布）/ 与共识差异 / 空头逻辑 + 反驳 / 核心赌点 / 可证伪条件 / 风险。
3. **公司快照（每只必带，放标题下）**：让读者只看推票就懂"主营 + 财报"、不必翻深报。固定三行——**主营**（分部占比一句）、**最近财报**（营收/利润/增速/毛利率/现金流，走一手三表）、**估值**（现价·PE·一致目标·评级）。模板见 `references` ⑥。财报数字回一手（美股 SEC EDGAR、港股注意币种），二手估算标"待核"；SPAC 无经营财报要写明。
4. **K 线按持仓期差异化**：短线 = 日K 约 1 月、中线 = 周K 约 6 月、长线 = 周K 约 3 年。`fetch_bars.py --interval 1wk --range 3y` 取周线；`kcharts.py` 传 `interval`(1D/1W) + `highlight:false`（推荐段不画复盘高亮）；周期矩阵见 `references` ⑤。
5. **三件套（每只都要）**：**① 进场时间/区间；② 止盈止损计算逻辑**（进场锚 × 分层比例，附风险收益比 RR）**；③ 可验证/可证伪条件**（带阈值、能用行情或财报判定）。默认分层标尺见 `references` ⑦：短线 +30%/−12%；中线 +80% 或卖方一致目标（孰近用孰）/ −20% 或 thesis 证伪；长线不设机械止盈（街目标仅作参考锚）、thesis 破坏才判错。
6. **组合层检查**：列各标的主导驱动力，明示是否押同一 beta（集中度风险）——别只看"不是同一只票"就当分散（例：NVDA/TSM/AI-SPAC 都押 AI capex 是同一 beta 的多种表达）。

## Step 6 · 交付前验收（硬门槛）

1. **图表**：`python scripts/validate_echarts.py <md>` 必须 0 failures（所有 ```echarts JSON 合法 + 日K 高亮标签落在 X 轴内）。
2. **口径**：文首数据口径齐全；A股/美股行业同时窗可比。
3. **写作红线自检**（见 `references/data-and-charts.md` 第三节）：无元话语/纠错痕迹、无"从 X 切到 Y"过度简化、Part4 是推票口径非深度报告口径、数字优先、红涨绿跌。跑一遍禁词/第一人称扫描（无"我/我们"、无翻译腔 calque 如"顺风/逆风/敞口"）。
4. **若写了 Part 5（Step 5B）**：每只都做过标的核验（无退市/借壳/拆股误判）；公司快照三行齐、财报回一手（二手标"待核"、SPAC 写明无经营财报）；K线周期与持仓期匹配且多年周K标签不糊；三件套（进场/止盈止损计算+RR/可证伪）齐；组合层检查点了 beta 集中度。
5. 用户改了口径/结论后，**先改方法论（本 skill），再改产物**；对单个 bad case 先归类再改规则，别只堵特例。

## 图表生成（复用轮子）

蜡烛图用 `scripts/kcharts.py`：`fetch_bars.py` 取好 bars → 写 `spec.json`（`highlight_start` + 每只 `title/sub/unit/bars`，另可选 `interval`(1D/1W) 与 `highlight`(bool)）→ `python scripts/kcharts.py spec.json` 打印 ```echarts 块 → 插到 md 对应位置。脚本按 `interval` 与 bar 数自动决定日期格式与标签密度（多年周K 用 YY/MM、每季一标签）；复盘段用日K + 高亮，推荐段（Step 5B）传 `highlight:false` 且按持仓期用 1D/1W。其余四类图（跨市场 bar / rebased 折线 / 行业全景 bar / 个股跌幅 bar）按 `references/data-and-charts.md` 模板手写 JSON，用 `validate_echarts.py` 校验。脚本仅用标准库、强制 UTF-8 输出。

## 边界

- 本 skill 专注**复盘**（验证已推的票，Part 1–4；Part 5 只把**已定清单**写成推荐段）。**不做全市场选股、不做深度覆盖/估值建模**——那些用你自己的选股与建模流程，本 skill 只吃清单、客观记录兑现结果。
- 复盘不改上周推票原文。
