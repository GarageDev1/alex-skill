# 取数配方 + 图表模板 + 写作红线（pitch-review 参考）

## 一、取数：哪个源取什么（真实、当次取，别用记忆里的旧数）

### 0. 唯一默认源 —— `scripts/fetch_bars.py`（Yahoo Finance，免 token）
本 skill 全部行情走这一个脚本，无需任何账号/密钥/私有服务器：
```
python scripts/fetch_bars.py --interval 1d  --range 1mo --spec ^GSPC ^SOX 2330.TW > spec.json   # 复盘日K
python scripts/fetch_bars.py --interval 1wk --range 3y  --spec NVDA TSM        > spec.json       # 长线周K
python scripts/fetch_bars.py --raw GLW MU 0700.HK > bars.json                                    # 只要 bars 自己组 spec
```
返回契约：`bars:[{t,o,h,l,c,v}]`，`t` 为 Unix 秒（UTC）。**区间涨跌基准**取"区间前最后一个收盘"（区间 6/29–7/8 → 基准 6/26 收盘）；美股当日未收盘则末点取上一交易日收盘，并在文首口径写明。

### 2. A 股申万一级行业区间涨跌（Yahoo 无，需补一个免费/自备源）
Yahoo 没有申万行业指数。用免费源（如 `akshare`，`pip install akshare`）或任何满足契约的结构化源取 31 个一级行业的基准日与末日收盘，算区间涨跌，与美股 GICS **同一口径同时窗**（如都 6/26→7/7）才能横向对比。
- 申万一级（SW2021）31 个行业代码 `801xxx.SI`：农林牧渔 801010 / 基础化工 801030 / 钢铁 801040 / 有色金属 801050 / 电子 801080 / 家用电器 801110 / 食品饮料 801120 / 纺织服饰 801130 / 轻工制造 801140 / 医药生物 801150 / 公用事业 801160 / 交通运输 801170 / 房地产 801180 / 商贸零售 801200 / 社会服务 801210 / 综合 801230 / 建筑材料 801710 / 建筑装饰 801720 / 电力设备 801730 / 国防军工 801740 / 计算机 801750 / 传媒 801760 / 通信 801770 / 银行 801780 / 非银金融 801790 / 汽车 801880 / 机械设备 801890 / 煤炭 801950 / 石油石化 801960 / 环保 801970 / 美容护理 801980。

### 3. 板块催化 / 事件 / 领涨领跌股 —— Web 搜索
搜"申万一级行业 涨跌幅 排名 <日期>"、"<板块> 大涨/大跌 原因"、"<市场> 板块轮动"。用于给数字配**机制与催化**（如创新药涨=医保目录改革、半导体跌=供给过剩+集体诉讼）。多家转引同一篇只算一个来源。

### 4. 财报三表 / 一致预期（Step 5B 公司快照用，一手优先）
- 免费一手源：**SEC EDGAR**（美股 10-K/10-Q/20-F，`https://data.sec.gov/`）、公司 IR 季报/新闻稿；A 股用免费源（如 akshare）或交易所披露；港股用交易所披露易。也可让 web-only 子 agent 带出处检索后回一手对账。
- **口径坑（用前必看）**：① 美股财年标签 = 期末所在年（NVDA FY 结 1 月，FY2026 标 "2026"）；② 港股币种要核对——部分公司（如腾讯）以 RMB 报账，别被标成 HKD 误导；③ 外国申报人（如 TSM，20-F 制）结构化源可能缺三表 → 退回公司官方季报口径，并在快照里标明来源。
- **毛利率自算**：`gross_profit / revenue`；增速用相邻财年/季报同比。二手转述的数字要回一手对账，对不上标"待核"。

### 5. 数据层契约（自包含 · 可替换）
本 skill 只依赖两个**字段契约**，换任何满足契约的公开源都能跑，默认实现即 `fetch_bars.py`：
- **行情源**：给定 `symbol, interval(1D/1W), count` → `bars:[{t,o,h,l,c,v}]`（`t`=Unix 秒 UTC）。默认 Yahoo；可替换为 yfinance / akshare / 券商 API 等。
- **财报源**：给定 `ticker` → 年度三表核心科目 + 现价/PE + 一致预期。默认 SEC EDGAR / 免费源 / 手填；可替换为任意结构化源。
- **红线**：任何 token/密钥/主机地址只在你本机的临时环境变量里出现，**绝不写进报告、skill 或会同步的目录**。可选走代理时用标准 `HTTPS_PROXY` 环境变量，脚本自动识别（不在文件里硬编码地址）。

## 二、五类 ECharts 模板（Obsidian ```echarts 代码块，中式红涨绿跌）

红涨 `#ee6666`，绿跌 `#3ba272`；正值红、负值绿。所有块交付前必须过 `validate_echarts.py`。

1. **跨市场区间涨跌幅 bar**（横向，最强→最弱排序）：一眼看清谁涨谁跌、分化多大。用于 Part2 开头。
2. **rebased 归一化折线**（基准=100）：多标的同图看相对强弱/背离（如宽基 vs 费半）。用于 Part2 美股、Part4 个股总览。
3. **日 K 蜡烛图 + markArea 高亮**：延伸约一个月，黄色阴影标复盘区间。用 `scripts/kcharts.py` 生成。用于 Part2 各市场关键指数、Part4 每只个股。
4. **行业全景 bar**（美股 GICS 11 行业 / A股申万31 行业，同口径）：证明"多方向扩散"，两图对称。用于 Part3。
5. **个股区间跌幅 bar**（按跌幅排序）：Part4 顶部总览，配 rebased 折线。

每张图后跟一句 `> **怎么读**：……`，用大白话点出这张图要说明什么判断（不是复述数字）。

## 三、写作红线（最终报告口径，逐条自检）

- **图文结合**：每个关键判断后就近配它的支撑图，别把图堆在一节末尾。
- **推票口径，不是深度报告口径**：Part4 复盘只引用"上周推票本身怎么说的"（推荐立场、买点区间、等回调、本股vs ADR 之类操作建议），**不要**搬深度报告的内部逻辑（风险编号、PE 阶梯、概率加权情景、护城河长篇）。
- **多方向，不过度简化**：轮动结论必须落到全市场板块数据；禁止"资金从 X 切到 Y"的单一对切叙事，除非数据只支持这一条。
- **无元话语 / 无纠错痕迹**：不写"需说明 / 需澄清 / 先要纠正一个说法 / 之前口径错了 / 需诚实标注"。有错直接改成干净陈述。不写对话式"你/我"。
- **不靠否定定义**：少用"这篇不讲 X 而是 Y""不是 A 而是 B"的自我纠正式框架；直接正面陈述结论。分析性否定（"我判断不会反转"）不在此列。
- **数字优先**：用"同比 +65.6%""区间 −6.8%"，不用"大幅""显著"。加粗只给结论和关键数。
- **时效性**：股价/财报/新闻/排名当次取最新；引用任何数字先核对时点。
- **口径统一**：文首写明区间基准与截止（如"6/26 收盘为基准，美股计至 7/7"）；A股/美股行业用同一时窗才可比。

## 四、下周推荐（Step 5B）配方

> Step 5B 把已定的标的清单写成推荐段。这一节是它的取数/画图/框架细则。

### ⑤ K 线周期矩阵（按持仓期取不同周期，别一律日K）

| 场景 | interval | count（约） | 跨度 | 高亮 |
|---|---|---|---|---|
| 复盘（Part2/Part4） | 1D | 25 | 约 1 月 | 是（复盘区间） |
| 推荐·短线 | 1D | 25 | 约 1 月 | 否 |
| 推荐·中线 | 1W | 30 | 约 6 月 | 否 |
| 推荐·长线 | 1W | 160 | 约 3 年 | 否 |

`fetch_bars.py` 用 `--interval 1wk --range 6mo/3y` 取周线；`kcharts.py` 按 `interval` 与 bar 数自动切日期格式（多年周K→YY/MM、每季一标签）。走势描述必须和 K 线对得上——"突破前高/新高"只在现价确实 ≥ 区间高点时才写。

### ⑥ 公司快照模板（Step 5B 每只必带，放标题下）

目的：读者只看推票就懂"主营 + 财报"，不必翻深报。固定三行 blockquote：

```
> **📇 公司快照**
> **主营**：<一句话业务本质 + 分部占比>（如 数据中心 AI 系统 + CUDA 生态，数据中心占 ~92%）
> **财报（源/币种）**：最近财年 营收 X(+y% YoY)｜净利/EPS｜毛利率 M%｜经营现金流；最近一季 关键分部/增速
> **估值**：现价｜P/E TTM ~a x / fwd ~b x｜卖方一致目标 $c(+d%)｜评级分布(买/持/卖)
```
- 财报走「一、4」的一手源；港股按实际币种标清；SPAC 写"无经营财报 + 标的财务待 S-4 一手披露"。
- 与正文"业务本质/估值"散文不重复算冗余——快照是**可扫读卡片**，散文是展开，各司其职。

### ⑦ 三件套 + 分层止盈止损标尺（本 skill 默认标尺）

每只推荐给三件套：

| 桶 | 时间窗 | 止盈 | 止损 | 止盈止损计算（②的算法） |
|---|---|---|---|---|
| 短线 | 1 周–6 月 | +30% | −12% | 进场锚 × (1±比例)；情绪票止损可用破位价（跌破 MAx/平台下沿）替代机械 −% |
| 中线 | 6 月–3 年 | +80% **或**卖方一致目标（孰近用孰） | −20% **或** thesis 证伪（证伪优先） | 目标价优先取一致目标；止损锚 −20% 或近期结构低 |
| 长线 | 3 年+ | 不设机械止盈（街目标仅作参考锚） | thesis 破坏才判错（可给结构低位作纪律风控参考，非机械止损） | 不算机械 RR；给"押注变量 + 破坏级证伪" |

- **① 进场时间/区间**：现价可分批 + 更优回踩位；带近端催化/财报日的注明（如"财报 07-22 前后据 cRPO 增速加减"）。
- **② 止盈止损计算（RR）**：进场锚 × 分层比例得止盈价/止损价，算 RR = 上行空间 ÷ 下行空间；长线不给机械 RR。
- **③ 可验证/可证伪条件**：带阈值、能用行情或财报判定（如"下季广告增速 <12%""数据中心收入连续两季环比下滑""跌破信托价 $10"）。
- **组合层检查**：列各标的主导驱动力，是否押同一 beta；同一 beta 的多种表达要明示为集中度风险。

### ⑧ 标的核验清单（Step 5B 第 1 步，防"拿旧认知当现状"）

对每个 ticker 逐条过，任一命中就切对应框架：
- **退市/被收购**？（老代码可能已消失，如 ChemoCentryx 被 Amgen 收购退市）→ 确认现在这个代码指向谁。
- **借壳/改名**？（SPAC 常见，如 CCXI = Churchill Capital XI 借壳 Agility Robotics）→ 主营写借壳后实体，估值用交易口径。
- **拆股**？（现价与印象差一个量级就查，如 NOW 5:1 拆股）→ 用拆后复权口径，快照注明。
- **SPAC / pre-close de-SPAC**？→ 无经营基本面，用事件驱动框架（信托价地板、S-4 披露、赎回/交割为证伪锚），不套经营公司模板；定位多为"观察/小仓博弈"而非"买入"。
- 核实工具：Web 搜索查最新事件，`fetch_bars.py` 拉现价对齐。
