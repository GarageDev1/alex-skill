# 估值镜头菜单

> Step 1 从这里选 ≥2 个镜头做三角验证。Step 4 落地。

## 镜头 × 适用类型
| 镜头 | 公式核心 | 最适用 | 陷阱 |
|---|---|---|---|
| **P/B-ROE** | justified P/B = (ROE−g)/(COE−g) | 重资产、周期、银行 | 账面快速复利时 trailing-book 会低估 |
| **P/E** | 价 = EPS × 目标 PE | 稳态盈利 | 周期底部 EPS≈0 失效;顶部低 PE 是陷阱 |
| **DCF** | Σ FCF/(1+WACC)^t + TV | 现金流可预测的稳态 | 终值假设主导、AI/周期股不稳 |
| **EV/Sales** | EV = Sales × 倍数 | 高增长无盈利 | 需配 path to profitability |
| **DDM** | 价 = Σ Div/(COE−g) | 高分红、银行险 | 不分红失效 |
| **EV/EBITDA** | EV = EBITDA × 倍数 | 跨资本结构比较 | D&A 重的资本密集型失真 |
| **SOTP** | Σ 各分部估值 | 多业务集团 | 分部口径/折价主观 |

## 三角验证原则
- **强制 ≥2 镜头**,禁止单镜头下结论。
- 重资产周期 / AI 重估标的:**P/B 和 P/E 必须都做**(这是真分歧,见 04-valuation reality check)。
- 多镜头给出区间,而非一个点;分歧大时,分歧本身就是研究结论(说明市场在赌什么)。

## ★★ 亏损 / 未盈利标的:主镜头切 EV/Sales,绝不把 P/E·P/B 留成一片 NA(铁律)
公司整体或某分部**当前亏损 / EBITDA 为负**时,P/E、P/B、EV/EBITDA 会变成 N/M 或 0——**这不是"无法估值",是选错了镜头**。规矩:
1. **主镜头自动切 EV/Sales**:亏损标的必须用 `EV = 营收 × 目标 EV/Sales` 拍出**实数**隐含价;P/E·P/B 退为诊断行(转正后再上),**不允许整张估值表只剩 NA/0 让读者无从判断**(实测翻车:用户看到全 NA 直接说"根本没法判断")。
2. **目标 EV/Sales 必须可锚、可解释**,三条定法(任选,优先 a):
   - a) **由盈利能力反推**:`EV/Sales = 目标 EV/EBITDA × 稳态 EBITDA margin`(把"将来能赚多少"翻译成现在的销售倍数,避免凭空拍 P/S)。这条让 EV/Sales 和 EV/EBITDA 内部自洽。
   - b) **同业 EV/Sales 光谱**:高增长 SaaS/平台/基建同业现倍数交叉。
   - c) **历史 EV/Sales 带**(若有上市历史)。
3. **配 path to profitability**:EV/Sales 必须挂一条"何年 EBITDA / 净利转正"的可证伪路径,否则是空中楼阁。
4. **分部亏损同理**:SOTP 里某分部亏损(如烧钱的 AI 段)→ 该分部用 EV/Sales,盈利分部(如现金牛)可用 EV/EBITDA,**混用但每个分部都出实数**。

## ★★ 前瞻估值必须「逐年」,绝不只拍单一终端年(铁律,SOTP 同样适用)
预测期内**每一年(如 2026E-2030E)都要出一档隐含价**,排成 NVDA 式逐年阶梯,而**不是只估一个终端年(如只拍 2030)再折现**。
- **为什么**:① 读者要看公司逐年"长进估值"的轨迹(现价 vs 各年隐含价,哪一年才追得上);② 单拍终端年 + 硬折现会系统性低估长久期成长(把多年价值压成一个被折掉的点);③ 逐年才能配合"12 个月目标价 = 最近前瞻年隐含价"。
- **做法**:每年一列:`各分部当年营收 × 当年目标倍数 → 分部 EV → Σ + 净现金 → ÷股本 = 当年隐含价`。目标倍数本身也**逐年演化**(高增长期高、随成熟 normalize),不是常数(见 04)。
- **SOTP 不是例外**:SOTP 也必须**逐年每段每年**出 EV,加总成逐年隐含价阶梯——**禁止只算终端年 SOTP**。
- **12M 目标价**取最近前瞻年(通常 NTM/次年)的隐含价;后续年份展示"成长进价"的路径。

## 各镜头落地公式
- **P/B–ROE**:`justified P/B = (ROE − g) / (COE − g)`;`目标价 = 目标P/B × BPS(某年)`。COE 用 CAPM(`Rf + β×ERP`)。ROE 用 ④ 段算出的可持续 ROE,不是峰值。
- **P/E**:`目标价 = 目标PE × EPS(某年)`。周期股用**归一化 EPS**(跨周期均值),不要用峰值 EPS × 低 PE(价值陷阱)。
- **DCF**:`EV = Σ FCFₜ/(1+WACC)ᵗ + TV/(1+WACC)ᴺ`;`股价 = (EV − 净债务)/股本`。**选用此镜头时必读 `lenses/dcf-lens.md`(深化版工艺,蒸馏自 equity-research 的 DCF 方法论)**:FCF 必须从主链 ③④ 接出(禁止另拍增速)、TV 双法互算、WACC 组件留痕、WACC×g + 物理锚×WACC 双敏感性矩阵、陷阱自检。DCF 是中位锚/交叉验证镜头,无固定权重。
- **EV/Sales**:`EV = 目标倍数 × 销售`,必须配 path to profitability(否则空中楼阁)。
- **DDM**:`价 = Σ Divₜ/(1+COE)ᵗ + 终值`。
- **EV/EBITDA**:`EV = 目标倍数 × EBITDA`;跨资本结构比较用。
- **SOTP**:各分部用各自最适镜头估值再加总,减去净债务/控股折价。

## 目标倍数怎么定(不许拍脑袋)
1. **历史估值带**(来自 `02-data-grounding`):目标倍数必须落在该标的历史 P/E·P/B·EV/EBITDA 的 min–max 带内;落在 median–p75 = 中性,贴 max = 乐观。
2. **可比公司**:同业当前倍数交叉验证。
3. **突破历史带**:必须写明 regime change 理由(如 AI 重估、商业模式切换),否则视为过度乐观。
4. 三情景对应三档倍数:Bear≈历史下沿 / Base≈median–p75 / Bull≈历史上沿或可比高位。
