# Step 6 — Reconcile + 决策 memo(模型与报告的统一结论层)

> **设置这一步的原因**:TSMC 首次覆盖报告给出 HOLD(USD 393,-0.7%),同期模型的 Base 情景却指向上行。两套假设独立生成且互不引用,导致结论冲突。本步要求同一 ticker 只保留一套估值结论。

## 触发场景(两种,处理不同)

| 场景 | 报告状态 | 本步做什么 |
|---|---|---|
| **A. 管线模式**(被 equity-research-v2 内嵌调用) | 报告尚未组装 | 只产决策 memo;报告组装相位会直接取模型数字,无旧账可对 |
| **B. 独立模式**(单独跑"建模 / 更新模型") | 同 ticker 已有首次覆盖报告 | 对账 + 决策 memo + **回填报告估值章节** |

## 开工硬性前置(两种模式都适用)

1. **现拉实时行情**:现价 / FX / ADR 价(如有双重上市)必须当场从可追溯公开行情源或用户指定数据源获取,**禁止无核验地沿用模型 input.json 里的旧价格**。memo 与回填的所有 upside 一律按经核验的最新价算,并写明时间戳与来源。
2. **双重上市必查溢价/折价**:ADR(或 H 股)实价 vs 本股×比例÷FX 的公允换算,差值 ≥5% 时必须在 memo「隐含价 vs 现价」段单列警示,并按两个口径分别给空间(TSMC 实例:ADR 溢价 16%,台北口径 +29% vs ADR 口径 +11%,评级因此拆成双轨)。
3. **读模型隐含价必须用计算值**:xlsx 若无公式缓存值(openpyxl 直读为空),先经本仓 `scripts/recalc.py --backend auto` 实算再取数,禁止手工照抄 sheet 里的中间假设自己重算。

## B 模式:对账(reconcile)

1. **找报告**:报告路径按显式参数 > `VALUATION_REPORT_PATH` > 不提供解析。未提供或文件不存在 → 跳过对账与回填,只出 memo;可选检索工具只能帮助定位,不能成为必需依赖。
2. **逐项对假设**:把报告正文里的可量化判断(分部占比走向、margin、目标倍数、关键物理量)与 `input.json` 逐项对比,产出**分歧表**:

   | 维度 | 报告说 | 模型 input.json | 谁对 / 怎么裁 |
   |---|---|---|---|

   裁决原则:**最新事实赢**(谁的数据更新、出处更硬,听谁的);裁不了的写明"真分歧 + 跟踪什么指标来裁"。
3. **回填报告估值章节**:报告的目标价 / 评级 / 情景表 / 估值汇总,**全部替换为模型的隐含价阶梯与三情景**;报告里旧的独立 DCF 加权目标价删除(不留两套数字)。回填时遵守报告写作规范(禁研究脚手架、code identifier、修复史——见全局 CLAUDE.md)。
4. 报告与 memo 中**禁止出现"模型 vs 报告打架"的过程叙述**——分歧表是给用户的工作文档(memo 附录),不是给报告读者的。

## 决策 memo(两种模式都强制)

输出目录按显式参数 > `VALUATION_OUTPUT_DIR` > 当前 workspace 的 `outputs/` 解析,落盘为 `<ticker>_决策memo.md`,一页,结构固定:

1. **Thesis 一句话**(来自定性研报的核心判断)
2. **隐含价 vs 现价**:Bear / Base / Bull 隐含价 + 概率 + 现价位置 → 上行/下行空间。**隐含价≠现价的差就是信号,不许为贴近现价回调假设**(见 SKILL.md ★★★)
3. **vs 卖方共识**:共识区间 + 本模型与共识的分歧点(哪条假设不同、凭什么)
4. **证伪触发器**:3-5 个可观测指标(对应「综合判断仪表盘」sheet),每个写明"出现什么读数 = thesis 被证伪 / 需要回模型改数重算"
5. **结论**:买入 / 持有 / 卖出 + 一句话理由

> memo 是"活模型"的对外接口:季报后更新模型 → 重出 memo → diff 上一版 memo 即知判断怎么变了。对接 investment-journal 时,thesis + 证伪触发器直接作为 judgment 的 `thesis` / `verify_criteria` 字段。

## 验收

- ☐ memo 五段齐全,每个隐含价能追溯到模型 sheet(连通性铁律延伸到 memo)
- ☐ B 模式:报告估值章节数字与模型完全一致,全文 grep 不到旧目标价
- ☐ 分歧表里每条裁决都有出处,没有"按感觉裁"
