# Step 2 — 数据落地

> 把 Step 1 定的锚 + 镜头需要的数据,全部落到标准 `<ticker>_input.json`,每个数字带完整 source 元数据。输入目录按显式参数 > `VALUATION_INPUT_DIR` > repo `out/` 解析。

## 取数主入口:标准 input.json 契约 ★
本 skill 不绑定任何仓外工具。公开公司披露、交易所/监管机构数据、行业机构材料、用户提供文件或可选外部数据工具,都先规范化为本页定义的 `input.json` 再进入建模。**铁律**:每个进模型的数字必带 `{source,date,url/file,rationale}`;无法追溯的材料只能用于提出待验证假设,不能直接成为硬编码输入。

## ★★ 标准取数流程:批量收集 → 并行深读 → 多家共识表(别只读1-2篇)
**绝不能只读 1-2 篇就拍数**(会以偏概全,真实翻车:只读 2 篇就误判"卖方止于2027",漏了 GS 2028/UBS 2030)。覆盖标的的卖方研报往往十几篇,必须榨干:
1. **批量收集全量**:扫描用户提供的报告目录并补充公开可获得材料;可选外部检索工具只负责加速定位,其输出仍必须落入相同 `input.json` 契约。
2. **并行子agent深读**:把 N 篇分批,**派多个并行子 agent**(Agent tool),每个读一批(图表是图 → 用 Read 渲染页面),各自回结构化预测:TP/评级/年限、逐年营收/OP/EPS、DRAM·NAND bit·价格、HBM TAM/份额/ASP、目标 P/E·P/B 及**倍数方法论**。
   - 路径含中文/特殊字符时 Read 会失败 → 先用 Python 拷成 ASCII 名再派 agent。
3. **汇成多家共识表**:逐年 × 逐家一张表(bit/价格/EPS/目标倍数/年限)。模型每个数标**共识区间 + 哪几家**;分歧大的(如价格 +110% vs +176%,口径差)给区间不给点。
4. **2028-2030**:多数家止于 2027-2028,少数(UBS/JPM)到 2030 —— 用它们当锚;再远的本模型周期判断,explicit 标注。
> 这套"批量+并行+共识"就是 Step 2 的正确姿势,不是可选项。

## ★ 单位与汇率约定(全模型统一)
- **基本面数据统一用 reporting currency = USD Billion($B)**:营收、利润、EBITDA、权益 —— 全部 $B,整表口径一致,不要 兆KRW 和 $B 混着。
- **外币标的折算(默认逐年均值汇率)**:历史年用**当年均值汇率**(Wind/BBG 利润表口径),前瞻年用现汇(无 FX 预测观点)。模型里放一行可见的「FX(KRW/USD,年均)」逐年,基本面 $B = 兆/当年FX。**单一汇率仅作 quick 兜底**(会扭曲历史年 USD 量级,如 2021 用现汇 1410 vs 真实 1144)。注:P/E·P/B 等倍数 FX 中性(分子分母同币),不受影响;受影响的是 $B 量级 + 市值。
- **币种边界**:估值在 reporting currency(USD)里做(净利$B×PE→市值$B→/股本→隐含价 USD);**只有与原始市场股价联动/对照时,才 ×FX 换回当地币种**(如 KRW)与实际成交价比。详见 `04-valuation.md`。
- FX 常数写进 `market.fx`,整表引用同一格。

## 必拉的两块历史底座 ★(链的现实校准器)

### A. 历史财务(锚定投影基年)
- 5 年三表关键科目 + **分部收入/毛利**(按 segment 拆,不要只有总数)
- **★ 必须含"最新一期实际报表"单独一列**(年中就是最新季报/半年报):把最新已实际公布的那期(如 1Q26)作为一列放进历史块——它是 base 上修/校准的硬锚(见 SKILL.md「最新季报作地板」)。规矩:① **YoY 用同比**(vs 去年同期 1Q25,不是 vs 上一年全年);② **单期不算 P/E·P/B**(一个季度的倍数无意义,市场估值那几行季报列留空);③ 单期分部若公司未拆,标"估"或留空、别硬编。
- **股本逐年 + 权益逐年**(各一行,不要单一值贯穿):公司可能增发/回购,EPS/BPS 必须按当年实际股本与权益算(详见 `04`「股本+权益必须逐年」)。分清**发行股 vs 基本流通股**(终端口径通常用基本流通股)。
- 用途:Step 3 的 ③营收/④利润/账面 投影起点 = 这里的真实值。**禁止 plug/反算基年。**

### ★★ 股价序列必须实拉,绝不手填/估一条曲线(数据铁律)
历史股价(月度/日度序列、年末价、年均价、现价)**必须从真实数据源抓**(tvremix `get_ohlcv` / global-stock-data / 交易所 API),**绝对禁止在脚本里手敲一条估计的价格曲线**。
- **实测翻车**:Hynix 模型的月度股价是硬编码的——2026-04 手填成 2.33M(真实 ~1.25M)、2025 年末填成 909k(真实 ~677k,909k 其实是 2026-01 底)。手填的曲线**既画错轨迹,又把历史 P/E·P/B 算错**(年末价错 → 倍数全错)。
- **现价与历史序列同源核对**:现价也用同一数据源抓,要和最近一根 K 线对得上(别一个 quote、一个 OHLCV 各说各话)。
- **年末价/年均价从抓来的序列算**(年末=该年最后一根收盘、年均=当年均值),不另手填。
- **工具**:`get_ohlcv(symbol, interval='1W' 或 '1D', count=N)`。**坑**:tvremix 的 `'1M'` 会被解析成 **1 分钟**(不是月线)→ 用日/周线抓、再 resample 成月度。
- 判断标准:模型里**任何一个价格数字**,都要能追到一次真实数据抓取;追不到 = 造假,必须重抓。

### B. 历史股价 + 历史估值带(校准目标倍数 + reality check)
- 历史股价序列(够长,覆盖至少一个完整周期)
- **★ 历史财务底座块每年同时带「年末价」+「年均价」两行**(都从同一条实拉月度序列算,不另手填):年末价对齐"那一刻"的 P/E·P/B,年均价平滑单点时点噪声(避免把某天的尖峰当成全年估值)。两个口径并列,读者一眼看出某年是"年末冲高"还是"全年都贵"。这条与 `04`「对照每年代表性股价」是同一原则在历史底座的落地。
- **历史估值带**:P/E、P/B、EV/EBITDA、ROE 的 min / 25% / median / 75% / max 分位 + 当前点
- **当前 TTM 锚(焊死交易终端)**:除历史年度倍数外,单列一个"当下"锚 = 现价 ÷ **TTM 实际 EPS**、现价 ÷ **最近一期 BPS**,数值要能**对齐 TradingView/Bloomberg 报价**(详见 `04`「当前倍数锚拍 TTM」)。这是模型与市场观测的对齐点。
- ★ **拉数时同时记下终端的现价 / TTM EPS / 最近一期 BPS / 终端报的 P/E·P/B**,作为当前锚的来源与复现依据。终端的 P/B 可能因"用年报权益 vs 用最近季度权益"出现两个值(口径不同),记清用了哪一期账面。
- 用途:Step 4 的目标倍数必须落在这个带内(突破要说明);隐含价阶梯叠回这个带做现实检验。
- **★ 落成独立「历史估值倍数」sheet**(放在「估值倍数假设」之前,先看数据再做假设):① 自身逐年 年末 P/E·P/B + **年内高/低 P/B 带**(由真实月度价算,不另拉);② 当下 TTM(对齐终端)+ forward 指引;③ **同业逐年 P/B + 当下 TTM/forward**(实拉、标源与抓取日;缺口径的年份标 n.a. 不硬补);④ **相对核心同行的 P/B 比值行**(如 海力士/三星)= 结构溢价的对账线,直接检验倍数页第二层取值。自身逐年值与外部数据商交叉核对(如 companiesmarketcap)写进来源注。

## 输出:input.json schema(草)
```json
{
  "hist_financials": { "2025A": { "revenue": ..., "segments": {...}, "net_income": ..., "equity": ... } },
  "hist_valuation_bands": { "PB": {"min":..,"p25":..,"median":..,"p75":..,"max":..,"current":..}, "PE": {...} },
  "market": { "price": ..., "shares_mn": ..., "net_debt": ..., "fx": ... },
  "anchor_inputs": { "<物理锚变量>": { "value":.., "source":.., "url":.., "date":.. } },
  "assumptions": { "bear": {...}, "base": {...}, "bull": {...} },
  "sources": { "<key>": { "source":.., "date":.., "url":.., "rationale":.. } }
}
```

## 历史估值带怎么算(从历史股价反推分位)
1. 取够长的历史**股价**序列(覆盖 ≥1 个完整周期)。
2. 配对每个时点的**每股指标**:trailing/forward EPS、BPS、EBITDA/股(从历史财务季度滚动)。
3. 逐时点算倍数:`P/E_t = 价_t / EPS_t`、`P/B_t = 价_t / BPS_t`、`EV/EBITDA_t`、`ROE_t`。
4. 对每个倍数序列取分位:`min / p25 / median / p75 / max + 当前点`,写进 `hist_valuation_bands`。
> 数据走 (d) 交易所/财务 MCP;周期股优先看 P/B(底部 EPS 失效)。

## input.json schema(关键字段)
- `hist_financials`:逐年三表关键科目 + **分部收入/毛利**(基年真实值,Step 3 投影起点)。
- `hist_valuation_bands`:各倍数 min/p25/median/p75/max/current。
- `market`:price / shares_mn / net_debt / fx。
- `anchor_inputs`:各物理锚变量 `{value, source, url, date}`(compute 类先引 `shared-base`,只补公司切片)。
- `assumptions`:bear/base/bull 三档,每档假设带 source + 触发条件。
- `sources`:每个 key `{source, date, url/file, rationale}`,由取数步骤直接落入。
> 校验:基年真实、倍数在带内、溯源齐全由 `scripts/validate_valuation.py` + Checks sheet 兜底。


## 数据源优先级(铁律)

行情 / 历史 K 线 / 三表 / 一致预期优先使用公司、交易所、监管机构或可信数据商的公开接口/文件;也可接受用户提供的数据。任何可选外部 skill/API 都只是同等输入适配器,不得成为唯一执行路径。发生数据源退路时,必须在 `input.json.sources` 写明实际来源、抓取日期与口径差异。
