# Step 4 — 多镜头估值

> 把 Step 3 算出的(粗颗粒)盈利,用 ≥2 个估值镜头折成隐含价,逐年排,对照每年代表性股价。
> **海拔**:目标倍数是**假设**,产出是**方向性判断**(便宜/合理/透支),不是精确目标价。
> **★ 例 ≠ 规范**:本文件是**底层方法论**(可迁移到任意标的)。文中所有带具体数字的例子(标"例:"或"如",多来自海力士 worked example `examples/build_hynix.py`)只是说明原则,**不是规范的一部分**——新标的一律按原则**现拉现算**(自己的历史峰值、自己的同业光谱、自己的对账线),禁止照抄例子数字。

> ★★ **拍目标倍数/盈利前先回看 SKILL.md「模型是干什么的」**:目的是**预测、指导投资**,不是拟合现价、不是跟共识。隐含价就该和现价不同(那是投资信号);**不怕拍出非主流的数,只要有事实+逻辑链**;最新数据(季报)指向更高/更低就照事实修。别为了"贴近现价"或"贴近共识"回调假设。

## ★★ 先判生死:亏损标的换镜头 + 任何前瞻估值必须逐年(两条总闸,先于下面所有细则)
1. **亏损 / EBITDA 为负 → 主镜头切 EV/Sales,绝不把 P/E·P/B 留成一片 NA**。亏损不是"无法估值",是选错镜头;目标 EV/Sales 优先用 `目标 EV/EBITDA × 稳态 EBITDA margin` 反推(与 EV/EBITDA 自洽),配 path to profitability。完整铁律见 `valuation-lenses.md`「亏损/未盈利标的」。
2. **前瞻估值必须逐年(2026E-2030E 每年一档隐含价),SOTP 同样逐年——禁止只拍单一终端年再折现**。单拍终端年会系统性低估长久期成长;逐年才能看"公司哪一年长进现价"。12M 目标价 = 最近前瞻年隐含价。完整铁律见 `valuation-lenses.md`「前瞻估值必须逐年」。

## ★ 默认:整体公司估值(把公司当一个整体)
**默认对整个公司估值,不做 SOTP 分部估值。** 投资人看的是一家公司,不是几个拼起来的段:
- **P/E 镜头**:市值 = 整体净利 × 目标 P/E → 股价 = 市值/股本(或直接 目标P/E × EPS)。
- **P/B 镜头**:股价 = 目标 P/B × BPS。
- 两个镜头并排,看分歧(这恰恰暴露周期股峰值盈利 vs 账面的张力,不用 SOTP 去抹平)。
- **SOTP 仅在特殊情况才用**:分部倍数差异极大、且用户明确要分部看时。否则一律整体估值。

## ★ 币种:估值在 USD 做,只在股价联动时换回当地币种
- 盈利/账面是 $B(见 `02` 单位约定)→ 隐含价先算成 **USD/股**(净利$B×PE×1e9/股本、或 PB×BPS_usd)。
- **对照实际股价**(原始市场是 KRW/其他)时,把 USD 隐含价 **×FX 换回当地币种** 再比;实际成交价保持原币种,不动。

## ★★ 当前倍数锚:拍 TTM(后视),不拍前瞻 —— 否则循环论证
建模时所在的"当前年",**当前倍数锚必须用 TTM(滚动12个月实际)**,不能用"现价 ÷ 自己预测的当年 EPS"。
- **为什么不能用前瞻**:模型存在的目的是让用户**独立判断**标的现在贵不贵。若"当前 P/E"= 现价 ÷ 我自己的前瞻 EPS,就成了"我看好→我拍高 EPS→算出 P/E 很低→模型说便宜→印证我看好"——**拿自己的乐观证明自己的乐观,循环论证**。前瞻 EPS 越乐观,当前 P/E 显得越便宜,买入信号越假。
- **TTM 的三个好处**:① 外生不可篡改(已发生的事实,观点掺不进);② 可复现(任何终端都能复核到同一个数);③ 不制造假缺口(也用现价,无"隐含价 vs 现价"的假象)。
- **焊死交易终端**:当前锚的 P/E、P/B 要能**对齐 TradingView/Bloomberg 等终端的报价**(现价 ÷ TTM EPS、现价 ÷ 最近一期 BPS)。落进 Excel 时把终端报的 P/E/P/B 作为来源标注,数值能复现。
- **前瞻 EPS 不是不用,是换位置用**:它只待在"未来"列(2027-2030 目标倍数 × 前瞻每股 → 隐含价);"当前年现价÷前瞻EPS"**降级为诊断行**(明确标注"前瞻诊断:以今日价你在为预测的当年盈利付几倍",不当作"当前 P/E")。

## ★★ 两张表对 P/E 的因果方向相反(事实 vs 判断)
- **历史/当前表**:**价是已知的 → P/E 被价倒推出来**(事实)。P/E = 当年实际股价 ÷ 当年实际 EPS,对齐终端年度列做回测。
- **预测表**:**P/E 是我们主动拍的输入 → 价被 P/E 算出来**(判断)。目标 P/E × 预测 EPS → 隐含价。目标倍数逐年演化本身就是分析师最核心的活儿。
- 一句话收口:**历史表 = 价→倒推 P/E(事实);预测表 = 拍 P/E→推价(判断)。** 别把两者混在一列。

## ★ 周期股的 TTM 陷阱(配 TTM 锚必须给的警告)
TTM P/E 对**周期股(内存/化工/航运…)**有经典陷阱:**盈利峰值年 TTM P/E 显得很低**(市场已 price-in 下滑)→ 低 P/E ≠ 便宜,反而是**顶部信号**;谷底年 TTM P/E 奇高才是真便宜。所以:
- TTM 锚的职责是**对齐现实 + 反映市场报价**,不是直接下"贵/便宜"结论。
- **便宜/透支的判断交给前瞻 P/B–ROE 路径**(账面不像谷底 EPS 那样失真)——这也是内存股看 P/B 优先于 P/E 的原因。

## ★★ 估值镜头的选择是业务判断,不是会计偏好(P/B 主线 vs P/E 主线 + 这一点要写进 Excel)
镜头选择回答的是一个**业务问题**:**这家公司"穿越周期持续存在的东西"是什么——盈利,还是资产?** 把那个东西当估值的分母。**这个"为什么"必须在 Excel 估值倍数页用一段文字、从业务层面写清楚**(不许只摆财务统计):
- **盈利本身是耐用资产 → P/E 主线(资本化盈利)**:有定价权 + 生态/客户锁定的公司(例:NVIDIA——fabless 轻资产、CUDA 生态把客户锁死),盈利可持续且增长——市场敢把"今年的盈利"外推很多年,所以用 P/E 资本化盈利。**P/E 高不是"贵"的同义词,是市场对盈利耐用性 + 增长的定价。**
- **重资产 ≠ 低 P/B——P/B 高低由"资产挣钱能力及其稳定性"决定,不由资产轻重决定**:例:TSMC 同样极重资产,但先进制程垄断 + 客户深度锁定让它的资产**穿越周期都挣高且稳的回报**,P/B 就可以很高(justified P/B=(ROE−g)/(COE−g) 的业务含义:分子就是资产的挣钱能力)。这类公司盈利也稳,P/E 同样可用。
- **商品型重资产 → P/B 主线**:产品无差异化、价格由行业供需定(内存/化工/航运/面板),公司对自己的盈利**没有定价权**——盈利是**周期状态量**(供需的函数,会大摆、会归零、会转负),资产/产能才是**穿越周期持续存在的结构存量**。给波动的盈利资本化没有意义(财务表现:周期顶峰值盈利算出低 P/E = 便宜假象;盈利回落同价 P/E 跳升;亏损年失效),给"产能 + 其穿越周期的挣钱能力"定价才有意义 → **P/B 主线,P/E 永远做支线体检(每个情景都做)**。
- **★ 镜头会随业务迁移——re-rating 的本质是市场换镜头**:当商品型公司长出 franchise 段(定制 + 认证壁垒 + 长约定价权,例:海力士的 HBM),业务从"商品"向"准代工/平台"迁移——这正是三层分解里**结构溢价**那一层的业务来源;若 franchise 完全证实,市场会逐步改用"资本化盈利"的镜头看它(P/B → P/E 迁移)。所以"该用哪个镜头"要写成**判断 + 触发条件**,不是永久标签。
- 配套财务事实(支撑、不替代业务逻辑):商品型公司账面逐年留存累积、单调增厚(低分红者尤甚)→ 跨周期可比;盈利经"留存→账面"和"ROE→应享倍数"两条路进入 P/B,信息不丢。
- **P/E 的正确位置 = 交叉验证(支线),且任何情景下都不可省略**:**不做平行隐含价**,而是取 **P/B 隐含价 ÷ 前瞻 EPS = 隐含 forward PE**,① 对照同业 forward PE 光谱看合不合理;② 标注盈利回落年同一隐含价对应的 PE 会跳升(暴露"低 PE 陷阱",反证为何不用 P/E 主线)。**★ 三情景架构下,「估值对比」每个情景 block 都要有一行隐含 forward P/E**——Bear/Base/Bull 各自的隐含价都要过一遍 P/E 体检(P/B 给答案,P/E 验答案),不是只验 Base。
- 对应到双镜头输出(下方「双镜头」节):**强周期标的市值/隐含价只从 P/B 主线出**,P/E 仅出一个"隐含 forward PE"诊断行,不参与市值平均。

## ★ 股本 + 权益必须「逐年」(可能增发/回购,单一值会算错倍数)
- **股本逐年一行**,不能用单一值贯穿,因为公司可能增发(稀释)或回购(增厚)。EPS/BPS 按当年实际股本计算;过往使用单一股本时,模型 P/B 与终端数据明显不一致。
- **发行股 vs 基本流通股(扣库存股)要分清**:终端的 P/E/P/B 通常用基本流通股,口径要对齐。若查下来"逐年基本不变"(无增发),那本身就是结论——**说明公司未稀释股东**,照实写一行近乎持平的值 + 注明"无增发"。
- **权益逐年取实际**(资产负债表 reporting currency),BPS = 当年权益 ÷ 当年股本;前瞻权益 = 上年 + 当年留存。

## ★ 历史年用「实际」倍数;forecast 倍数必须「逐年演化」(这是判断核心)
- **历史年 + 当前年(有实时股价的)**:P/E、P/B 直接用**实际/市场价反推**(当年价/EPS、当年价/BPS)——已知事实,**不要对当前年硬套目标倍数**。
  - ★ 易错:若对"当前年"(如建模时所在的 2026)硬套一个目标倍数算隐含价,会造出"隐含价 ~100万 vs 现价 ~236万"的**假缺口**,看着像市场疯了,其实是你给当前年拍的倍数离市场太远。我们就在这一年、有现价,**直接用现价反推当前年的 P/E、P/B**。
- **倍数路径方向**:高成长标的**当前市场已给高倍数**,forecast 倍数应从这个高位**随成长成熟逐年 normalize 下行**(P/E 高→低、P/B 跟 ROE 回落)——**别搞反成低→高**。
- **forecast 年**:倍数**绝不能逐年定一个常数**——倍数怎么排,本身就是分析师最核心的判断。两条铁律:
  - **P/B 跟 ROE 走**:`justified P/B ≈ (ROE − g)/(COE − g)`。ROE 从峰值 normalize → P/B 必须同步从高位回落(如 ROE 79%→14%,P/B 从 ~3x→~1.2x)。不是平的。
  - **P/E 做周期反转**:周期峰值盈利年给**低** P/E(市场知道盈利要塌,如 5-8x);normalize/底部年给**高** P/E。和 EPS 反向。
  - 每年的目标倍数是**带逻辑的逐年输入**(挂 ROE / 周期位置),在可见逻辑列写清"这年为什么是这个倍数"。
- 这样估值表能看到:历史真实倍数带 + 未来**随基本面演化**的倍数路径,而不是一条假装稳定的平线。

## ★ 双镜头的输出(隐含价 + 市值)要两镜头各出一个,别只用一个
> **★ 强周期重资产例外**:对内存/化工等 P/B-主线标的(见上「P/B 主线、P/E 支线」),**不走平行双镜头**——市值/隐含价只从 P/B 主线出,P/E 仅出一个"隐含 forward PE"诊断行。下面这段平行双镜头规则适用于**非强周期、P/E 与 P/B 同等可信**的标的。

使用 P/E、P/B 两种方法时,隐含股价和隐含市值都要按两种方法分别推导并列展示。不得同时展示两种隐含价,却只用 P/B 计算市值。
- **隐含价**:P/E 隐含价(目标P/E×EPS)、P/B 隐含价(目标P/B×BPS),两行,标清哪个镜头。
- **市值**:同样两行——市值(P/E镜头)= P/E隐含价×股本、市值(P/B镜头)= P/B隐含价×股本。要一个中性值就取两镜头均值,但**两个分镜头的市值都要留着让读者看分歧**(三角验证的意义就在于看两镜头差多少)。
- **upside**:vs 现价的隐含涨跌幅也两镜头各算一个。

## ★★ 估值倍数对照 sheet(目标倍数的唯一来源 + 放进参照系)
目标倍数不能孤立地拍,也不该埋在利润/收入假设里。**单开一张「估值倍数假设」sheet,它是前瞻目标 P/E·P/B 的唯一来源**——「估值」表的前瞻倍数**引用这张表**,而不是自己拍(「估值」表只负责"套用倍数 × 前瞻每股 → 隐含价")。对应地,利润率/收入假设收敛进「**利润与收入假设**」(连同它驱动的基本面 EPS/BPS),**不管估值倍数**。

**★ sheet 排成只向前引用的 DAG(见 `05` §1)**:`历史估值倍数(数据底座: 自身历史带 + 当下 TTM + 同业 + 相对比值, 全上游/实拉)→ 估值倍数假设(前置·自包含: 方法论 + 三层常数 + 三案情绪值/目标P/B 矩阵——三案倍数在这里拍)→ 情景切换(情绪值杠杆引倍数页三案行做切换, 出当前案目标P/B)→ 业务链 → 利润与收入假设(出 EPS/BPS)→ 情景估值(当前案目标P/B × 前瞻BPS → 隐含价)→ 估值对比`。
- **倍数页前置的纪律**:它**不得引用任何下游表**。代价:标的自身 forward PE(= 现价 ÷ 模型 EPS,要用下游)的**证据行放「情景估值」的 P/E 交叉验证块**,倍数页光谱里只留 peer(纯输入)+ 标的 TTM(引历史财务,上游)+ 一句指引。
- 逻辑顺序由此变成:**先设置三案倍数(倍数页)→ 切换(情景切换)→ 业务链 → 套用(情景估值)**。

**★ 组织成上下两半(既给证据、又给预测)**:
- **上半【证据】= 市场现在把它当什么定价**:
  - **Forward PE 横向对照表**:将标的与三类对象比较:高确定性成长公司(产业链龙头/平台)、标的所在市场与美股大盘、最直接可比同行,判断市场当前的定价类型。具体对象和数据按标的所在产业链当次获取(例:海力士对照 AI 半导体、S&P/KOSPI 和内存同业)。Forward PE 统一使用下一财年(NTM)共识口径。peer 取当次获取的单一可比值;标的使用 `现价 ÷ 模型下一财年前瞻 EPS` 计算可比值。
  - **加一列 TTM 对照**:周期股盈利暴涨时 TTM 失真(同业 TTM 可数倍于 forward)——TTM 列正好展示"盈利暴涨 → forward 压缩"机制,解释标的为何 forward 便宜。
  - 标的 forward PE **逐年**小块(FY26E/FY27E…)+ re-rating 判断 + 自身历史倍数带(看清 TTM 为何虚高)。
  - 这半部还兼做 **P/E 支线**:标的 forward PE 逐年 + 放进光谱 = P/E 交叉验证的证据(强周期标的 P/E 只做支线,见上「P/B 主线、P/E 支线」)。

- **下半【预测】= 用三层分解确定目标 P/B(GS 式)**:
  目标 P/B 不直接填入单一倍数,而是分解成三层相乘,每层都有独立来源且可对账(参考 GS/MS 做法):
  ```
  目标 P/B = 历史周期峰值 P/B  ×  结构溢价  ×  情绪值
            (该股【过去】最强周期    (本轮结构性重估幅度,    (周期/情绪位置,
             实际到过的 P/B 峰值)     锚同业相对法)          记分卡驱动)
  ```
  - **第一层 历史周期峰值**:取该股在**过去**最强周期里**实际**到过的 P/B 峰值(MS 等给"历史周期峰值"口径,如 Hynix 2.7x)。★ **绝不能用当下/最近(已被本轮重估抬高)的倍数当锚**——它已含结构溢价 + 情绪,再乘=双重计算。要用"没有本轮 re-rating 时它该值多少"。
  - **第二层 结构溢价**:本轮**结构性(非周期)**的重估幅度,锚**同业相对法**(如 GS:海力士相对三星的历史平均溢价 **30%**→ ×1.30)或可比公司光谱。慢变、半硬。
  - **第三层 情绪值**:周期/情绪位置。**1.0 = 市场付足"峰值×溢价";>1.0 = FOMO 超涨(付得比历史峰值×溢价还高);<1.0 = 下行周期折价。** 最软,由**记分卡**打分映射(见下),不是拍。
  - **★ 一致性检验(必做)**:三层相乘**应能复现该股最近的实际倍数**(如 2.7×1.30×1.11 ≈ 3.9x = Hynix 2025末实际 P/B)。复现不了 = 某层口径错,回去校。
  - **目标 P/B = 三层公式算出**(不是蓝字硬填);只有第一、二层常数 + 第三层情绪值是输入。这一行喂「估值」。
  - **非共识(B)在框架里有清晰位置**:给高于卖方的目标倍数 = **情绪值 > 1.0**(押本轮比历史任何周期都强)。是有判断力的拍法,但必须挂 kill 信号(见下情绪持续性)。

- **★ 情绪值的持续性(最易忽略、却决定 B 成败)**:情绪值 >1.0 的"超涨"能撑多久,**必须用历史类比 + 领先信号定,不能默认它一直撑着**:
  - **历史类比(需实时检索)**:查同类资产高情绪定价的持续期。历史上内存周期顶部通常不超过 4 个季度;Cisco 在 dotcom 期间溢价扩张约 3 年,之后 2 年内下跌 88%,25 年后仍未回到原高点。本轮如有售罄和长约提高可见度,持续期可高于常规周期(如 18-24 个月),但仍需设置边界。重资产公司的账面价值可提供部分下行支撑。
  - **情绪值路径**:据此排——可见度内(如 FY26-27)情绪 >1.0,可见度耗尽后(FY28+)跌破 1.0、向"峰值×溢价"之下收敛。
  - **监测"情绪还能撑多久" = 盯领先信号,不盯股价**:如"明年是否还售罄"(最强先行)、"ASP 首次环比跌"、"竞品认证通过(结构溢价缩)"、"下游 capex 指引转下修"。任一翻 → 下调情绪值。这些就是记分卡的衰减扳机。

- **★ 情绪值:不必精确量化——预测引擎是『错位 + 催化剂』,情绪只做 timing 确认 + 过热刹车**:FOMO 超涨的精确幅度本就不可测(2025 年没人能算出 Hynix 会冲 6.8x P/B)。能在拐点**之前**看见行情的,是两样当时就观测得到的东西:**估值错位**(市场按什么定价 vs 基本面该给什么 → GAP%,可量化)和**催化剂**(什么会逼市场闭合 GAP,可清单化)。情绪值只需**定性档位**(萌芽/启动/过热/退潮 → 映射粗档,如 启动≈1.0-1.1 / 过热>1.15 / 退潮<1.0),由「综合判断仪表盘」(`05` 4c)的读数支撑——不追求算准,只回答两个问题:① 开始动了没(动量/放量确认);② 过热了没(现价倍数 ≫ 基本面该给 → 该兑现不该追)。
- **★ 量价情绪只当温度计,不当估值输入(循环论证红线)**:动量/换手率/乖离/散户净买入(韩股 KRX 有散户流披露)可以算一个客观的"市场情绪温度"——有理论撑(Baker-Wurgler 情绪指数含换手率;且**高情绪预示低后续回报** = 天生的反向/过热指标)。但**绝不能把量价拟合进目标倍数**:情绪从现价量价来、再推隐含价 → 隐含价追着现价跑,信号归零(同"现价÷自己前瞻EPS"的循环论证)。正确用法:量价温度 vs 基本面判断**对比看背离**(量价 ≫ 基本面 = FOMO 脆弱该收;量价 ≪ 基本面 = 过度悲观买点),并作衰减扳机的最早领先信号。
- **★ 仪表盘的验收标准 = 回测拐点**:把「综合判断仪表盘」放回标的上一轮行情的拐点,它必须能让你**当时**就看到那波(A 基本面拐点强 + B 错位大 + C 催化剂兑现中 + D 情绪刚启动 → 强烈看多)。做不到 = 仪表盘没装上预测引擎,只是事后记录。

- **peer 数据实拉**:TTM/forward 倍数、历史峰值 P/B 与结构溢价必须来自可追溯的公开数据、用户提供文件或可选数据工具,统一写入 `input.json.sources`;**绝不用示意数字当数据**。
- **判断标准**:① 目标 P/B 由三层公式计算(修改情绪值时结果会变),不直接填入数值;② 三层相乘能复现最近实际倍数(通过一致性检验);③ 情绪值有记分卡依据,持续性有历史类比;④ 强周期标的 P/E 只作辅助对照(隐含 forward PE),市值由 P/B 推导。

## 硬动作
1. **≥2 镜头三角验证**(整体口径)。禁止单镜头下结论。重资产周期/AI 重估标的:P/B-ROE 和 P/E 两个都做。
2. **逐年隐含价(NVDA 式)**:每年一列 —— 目标 P/E × 当年 EPS、目标 P/B × 当年 BPS → 当年隐含股价。
3. **★ 预测表必须回溯过去 + 对照每年代表性股价**:
   - 估值表不要只排未来。**2021 起的历史年份也排上**(过去盈利/账面是实际或可往上拍的估计)。
   - 对照价不得全部使用同一个当前价。每年使用该年的代表性股价:① 年末收盘价 ② 年均价(月度均值)。历史年份用真实成交价,通过隐含价与实际价的差异回测模型;未来年份与当前价比较。
   - 这比"模型 vs 单一现价"信息量大得多:看到模型隐含价的逐年轨迹 vs 股票实际怎么走。

## 三情景(Bear/Base/Bull)★★ —— 全局「情景切换」驱动全链 + 「估值对比」恒常并排
标准架构 = **驱动和展示分开**:开关驱动整条明细链变档,对比 sheet 三列恒常可见。
- **★「情景切换」sheet(放在卖方研报共识之后、链的最前面)= 全模型唯一的情景参数库**:① 切换开关(下拉 Bear/Base/Bull,**默认 Base = 主线**)是**唯一入口**;案序号由它**派生**(`=IF(开关="Bear",1,IF(开关="Base",2,3))`——**用嵌套 IF,禁用 MATCH+数组常量 `{"..."}`**,后者在 WPS/部分 locale 下报错,整条 INDEX 链会失效);② 每案一句**触发条件**(什么发生 = 落进这档);③ **参数矩阵**:每个可翻档杠杆 × 三案(全蓝字,只在这页改)+ 一行『当前案』(`=INDEX(三案行, 案序号)`)。**矩阵布局**:杠杆名独立一行作组标题(带底色),下面缩进挂 Bear / Base / Bull / 当前案 四行——不要把杠杆名和 Bear 挤在同一行。**说明列分两层**:组标题行放**指标本体描述**(这变量是什么、为什么重要、共用锚是什么);Bear/Base/Bull **各自行**放**该案的具体故事**(事件→时点→数字路径→出处)——各放各行,不合并跨组、不堆一格。
- **★ 明细链吃『当前案』行**:物理锚增速 / 收入强度 / 价格周期 / 段利润率 / 情绪值 等可翻档输入 = 引『情景切换』当前案行 → 切开关,**整条明细链(锚→测算→利润→倍数→估值)整体变档**。「估值」页改名「**情景估值**」(输出 = 当前案的逐年隐含价)。
- **★ 杠杆选择原则(模型自由判断,不写死清单)**:只翻 **load-bearing 的少数杠杆(~5-7 个)**,其余三案共用(跟 Base)。判断标准:「这个假设翻档会显著改变隐含价、且三案故事里它**真的**不同」。典型集(供参考):物理锚增速路径 / 收入强度·份额 / 价格周期(涨幅或 normalize 时点)/ 段利润率 / 情绪值路径。翻太多 = 参数矩阵爆炸、故事失焦。
- **★「估值对比」sheet(三案主输出)**:顶部**三列汇总**(每案 触发条件 / 目标年 NI / 总收入 / 目标倍数 / 隐含价 / vs 现价 / 市值)+ 下方**每案一个逐年 block**(NVDA worked model 式:锚→分部收入→NI→权益→BPS→倍数→逐年隐含价,全公式)。**恒常三列并排、不随开关变**。
  - **★ 防污染铁律**:对比列**绝不引用明细链上会随开关变的格**(否则被当前案污染)——case 列只引『情景切换』**矩阵行**(Bear/Base/Bull 各自的行,不是当前案行)+ 链上**未翻档**的行(强度/bit/转换率等跟 Base 的)+ 静态历史锚。Base 列与明细链同公式同输入 → 开关=Base 时两者完全一致(同构自检)。
- **下拉规则(修正)**:下拉用于**驱动链**(全局 case switch)是机构模型标配,OK;禁止的是**只用下拉展示、不配恒常并排的对比 sheet**(把价差藏掉、截图丢档)。
- **情景必须和业务挂钩——盈利由业务杠杆沿同一条链算出,绝不手拍 NI**;三案共用同一物理锚起点(改锚,三案全动;连通性测试对情景同样适用)。
- **每档是一个连贯的故事 + 触发条件**,不是 ±20% 调数字;杠杆值带 source(共识哪家/管理层口径/最新季报)。
- **★ 情景说明的可读性标准**:每个杠杆的说明 = **机制一句(这变量是什么、为什么重要)+ 每案一条具体故事(事件→时点→数字路径→出处)**。**禁缩写串**(如"Bear=2027早松提前跌价"这种没人看得懂的压缩话——实测被用户点名"可读性太差")。写之前用研报库语义检索 + web search 把每案故事做实(具体券商口径/最新数据),再加自己的判断;版面不够就合并单元格给足空间,不是把话压短。
- **★ 已实现的最新业绩是 Bear 的地板**:最新季报已发生的盈利,Bear 档不能低于其年化(已经赚的不会蒸发)。
- **★ 情景矩阵与对比块都要配历史实际列(2021-2025),不留空**(03「假设行配历史实际列」在情景架构的延伸):① 情景矩阵每个杠杆的历史列填**实际值,三案同值**(历史只有一个,n.m. 的标 n.m.)——读者一眼看到"前瞻拍的数 vs 历史实际"的落差;② 情绪值的历史列可**公式反推**(= 实际 P/B ÷ 峰值×溢价),一致性检验直接可见;③ **「估值对比」每案 block 的历史列用同一条链跑实际值**——隐含价历史列应**≈实际年末价**(vs 实际年末价 行 ≈0%),这就是整条链的**内置回测**,验证"这套公式放回过去能不能复现现实"。
- 输出 = 三档隐含价 + vs 现价(risk-reward 不对称一行收口);P/B 主线标的只跑 P/B 镜头(P/E 支线规则照旧)。

## Sensitivity 网格已废除(职能由三情景架构承担)
