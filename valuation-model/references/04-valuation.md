# Step 4 — 使用多种方法估值

> 使用至少两种估值方法,将 Step 3 算出的盈利转换为逐年隐含价,并与各年代表性股价对照。
> 目标倍数是**假设**,产出是**方向性判断**(便宜/合理/透支),不是精确目标价。
> 本文件说明可迁移到不同标的的方法。文中标为"例:"或"如"的数字只用于解释原则。新标的必须重新获取自身历史倍数、同业数据和核对基准，不得照抄示例数字。

> 设置目标倍数和盈利假设前,先回看 SKILL.md 中的模型用途。模型用于预测并支持投资决策,不用于拟合现价或复述共识。隐含价可以与现价不同,但非共识假设必须有事实和完整推导支撑。最新数据指向更高或更低的价值时,按数据更新假设,不为贴近现价或共识而回调。

## ★★ 两项前置要求:亏损标的更换估值方法,前瞻估值按年计算
1. **亏损 / EBITDA 为负 → 主要估值方法切 EV/Sales,绝不把 P/E·P/B 留成一片 NA**。亏损不是"无法估值",是选错估值方法;目标 EV/Sales 优先用 `目标 EV/EBITDA × 稳态 EBITDA margin` 反推(与 EV/EBITDA 自洽),配 path to profitability。完整要求见 `valuation-lenses.md`「亏损/未盈利标的」。
2. **前瞻估值必须逐年(2026E-2030E 每年一档隐含价),SOTP 同样逐年,不得只设置单一终端年再折现**。单一终端年可能系统性低估长久期成长;逐年计算可以判断哪一年的业绩能够支持现价。12M 目标价 = 最近前瞻年隐含价。完整要求见 `valuation-lenses.md`「前瞻估值必须逐年」。

## ★ 默认:整体公司估值(把公司当一个整体)
**默认对整个公司估值,不做 SOTP 分部估值。** 投资人看的是一家公司,不是几个拼起来的段:
- **P/E 估值方法**:市值 = 整体净利 × 目标 P/E → 股价 = 市值/股本(或直接 目标P/E × EPS)。
- **P/B 估值方法**:股价 = 目标 P/B × BPS。
- 两个估值方法并排,看分歧(这恰恰暴露周期股峰值盈利 vs 账面的张力,不用 SOTP 去抹平)。
- **SOTP 仅在特殊情况才用**:分部倍数差异极大、且用户明确要分部看时。否则一律整体估值。

## ★ 币种:估值在 USD 做,只在股价联动时换回当地币种
- 盈利/账面是 $B(见 `02` 单位约定)→ 隐含价先算成 **USD/股**(净利$B×PE×1e9/股本、或 PB×BPS_usd)。
- **对照实际股价**(原始市场是 KRW/其他)时,把 USD 隐含价 **×FX 换回当地币种** 再比;实际成交价保持原币种,不动。

## ★★ 当前倍数使用 TTM,避免前瞻假设形成循环论证
建模时所在的"当前年",**当前倍数基准必须用 TTM(滚动12个月实际)**,不能用"现价 ÷ 自己预测的当年 EPS"。
- **为什么不能用前瞻**:模型用于独立判断当前估值。若"当前 P/E"=现价÷模型前瞻 EPS,更高的 EPS 假设会直接压低当前 P/E,再被用来证明标的便宜,形成循环论证。前瞻 EPS 越乐观,这一偏差越大。
- **TTM 的三个好处**:① 外生不可篡改(已发生的事实,观点掺不进);② 可复现(任何终端都能复核到同一个数);③ 不制造假缺口(也用现价,无"隐含价 vs 现价"的假象)。
- **与交易终端核对**:当前估值的 P/E、P/B 要能**对齐 TradingView/Bloomberg 等终端的报价**(现价 ÷ TTM EPS、现价 ÷ 最近一期 BPS)。写入 Excel 时标注终端 P/E/P/B 的来源,并确保数值可复现。
- **前瞻 EPS 不是不用,是换位置用**:它只待在"未来"列(2027-2030 目标倍数 × 前瞻每股 → 隐含价);"当前年现价÷前瞻EPS"**降级为诊断行**(明确标注"前瞻诊断:以今日价你在为预测的当年盈利付几倍",不当作"当前 P/E")。

## ★★ 两张表对 P/E 的因果方向相反(事实 vs 判断)
- **历史/当前表**:**价是已知的 → P/E 被价倒推出来**(事实)。P/E = 当年实际股价 ÷ 当年实际 EPS,对齐终端年度列做回测。
- **预测表**:**P/E 是主动设置的输入 → 价由 P/E 算出**(判断)。目标 P/E × 预测 EPS → 隐含价。逐年目标倍数需要单独说明依据。
- **口径汇总**:**历史表 = 价→倒推 P/E(事实);预测表 = 设置 P/E→推价(判断)。** 两者不得混在一列。

## ★ 周期股的 TTM 陷阱(配 TTM 基准必须给的警告)
TTM P/E 对**周期股(内存/化工/航运…)**有经典陷阱:**盈利峰值年 TTM P/E 显得很低**(市场已 price-in 下滑)→ 低 P/E ≠ 便宜,反而是**顶部信号**;谷底年 TTM P/E 奇高才是真便宜。所以:
- TTM 基准的职责是**对齐现实 + 反映市场报价**,不是直接下"贵/便宜"结论。
- **便宜/透支的判断交给前瞻 P/B–ROE 路径**(账面不像谷底 EPS 那样失真):这也是内存股看 P/B 优先于 P/E 的原因。

## ★★ 估值方法的选择是业务判断,不是会计偏好(P/B 主要估值方法 vs P/E 主要估值方法 + 这一点要写进 Excel)
估值方法取决于归一化盈利和资产基础中哪一个更稳定。选择理由必须在 Excel 估值倍数页从业务层面说明,不能只列财务统计:
- **盈利稳定且可持续 → P/E 主要估值方法(资本化盈利)**:有定价权和较高客户转换成本的公司(例:NVIDIA的 CUDA 生态),盈利具有较高持续性,可以用 P/E 资本化。**较高 P/E 反映市场对盈利持续性和增长的定价,不能单独据此判断高估。**
- **重资产 ≠ 低 P/B:P/B 高低由"资产挣钱能力及其稳定性"决定,不由资产轻重决定**:例:TSMC 同样极重资产,但先进制程垄断 + 客户深度锁定让它的资产**穿越周期都挣高且稳的回报**,P/B 就可以很高(justified P/B=(ROE−g)/(COE−g) 的业务含义:分子就是资产的挣钱能力)。这类公司盈利也稳,P/E 同样可用。
- **商品型重资产 → P/B 主要估值方法**:产品差异有限、价格由行业供需决定的公司(内存/化工/航运/面板),盈利随周期大幅波动,可能降至零或转负;资产和产能相对稳定。此时峰值盈利会使 P/E 看似偏低,盈利回落又会使 P/E 快速上升,因此以 P/B 为主,P/E 用于各情景的交叉验证。
- **业务结构变化时重新选择估值方法**:当商品型公司增加定制、认证壁垒和长约定价业务(如海力士 HBM),结构性溢价可能上升。若该业务的盈利持续性得到验证,估值可以逐步从 P/B 转向 P/E。因此估值方法需要同时写明判断依据和切换条件。
- 配套财务事实(支撑、不替代业务逻辑):商品型公司账面逐年留存累积、单调增厚(低分红者尤甚)→ 跨周期可比;盈利经"留存→账面"和"ROE→应享倍数"两条路进入 P/B,信息不丢。
- **P/E 用于交叉核对(辅助方法),且任何情景下都不可省略**:**不单独计算 P/E 隐含价**,而是计算 **P/B 隐含价 ÷ 前瞻 EPS = 隐含 forward P/E**。将结果与同业 forward P/E 区间对照,并标注盈利回落年份对应的 P/E 上升幅度,用于识别"低 P/E 陷阱"。**★ 三种情景下,「估值对比」的每个情景区块都要列出隐含 forward P/E**,分别核对 Bear、Base 和 Bull,不能只核对 Base。
- 对应到两种估值方法输出(下方「两种估值方法」节):**强周期标的市值/隐含价只从 P/B 主要估值方法出**,P/E 仅出一个"隐含 forward PE"诊断行,不参与市值平均。

## ★ 股本 + 权益必须「逐年」(可能增发/回购,单一值会算错倍数)
- **股本逐年一行**,不能用单一值贯穿,因为公司可能增发(稀释)或回购(增厚)。EPS/BPS 按当年实际股本计算;过往使用单一股本时,模型 P/B 与终端数据明显不一致。
- **发行股 vs 基本流通股(扣库存股)要分清**:终端的 P/E/P/B 通常用基本流通股,口径要对齐。若查下来"逐年基本不变"(无增发),那本身就是结论:**说明公司未稀释股东**,照实写一行近乎持平的值 + 注明"无增发"。
- **权益逐年取实际**(资产负债表 reporting currency),BPS = 当年权益 ÷ 当年股本;前瞻权益 = 上年 + 当年留存。

## ★ 历史年用「实际」倍数;forecast 倍数必须「逐年演化」(这是判断核心)
- **历史年 + 当前年(有实时股价的)**:P/E、P/B 直接用**实际/市场价反推**(当年价/EPS、当年价/BPS):已知事实,**不要对当前年硬套目标倍数**。
  - ★ 易错:若对"当前年"(如建模时所在的 2026)应用目标倍数计算隐含价,可能产生"隐含价 ~100万 vs 现价 ~236万"的虚假差额。当前年已有市场价格,应**直接用现价反推当前年的 P/E、P/B**。
- **倍数路径方向**:高成长公司当前通常对应较高倍数。forecast 倍数应随公司增长趋于成熟而逐年 normalize 下行(P/E 高→低、P/B 随 ROE 回落),不得设置为由低到高。
- **forecast 年**:倍数不得固定不变。逐年倍数需要满足两项要求:
  - **P/B 跟 ROE 走**:`justified P/B ≈ (ROE − g)/(COE − g)`。ROE 从峰值 normalize → P/B 必须同步从高位回落(如 ROE 79%→14%,P/B 从 ~3x→~1.2x)。不是平的。
  - **P/E 做周期反转**:周期峰值盈利年给**低** P/E(市场知道盈利要塌,如 5-8x);normalize/底部年给**高** P/E。和 EPS 反向。
  - 每年的目标倍数是**带逻辑的逐年输入**(挂 ROE / 周期位置),在可见逻辑列写清"这年为什么是这个倍数"。
- 这样估值表能看到:历史真实倍数带 + 未来**随基本面演化**的倍数路径,而不是一条假装稳定的平线。

## ★ 两种估值方法的输出(隐含价 + 市值)要两种估值方法各出一个,别只用一个
> **★ 强周期重资产例外**:对内存/化工等 P/B-主要方法标的(见上「P/B 主要估值方法、P/E 辅助方法」),**不走平行两种估值方法**:市值/隐含价只从 P/B 主要估值方法出,P/E 仅出一个"隐含 forward PE"诊断行。下面这段平行两种估值方法规则适用于**非强周期、P/E 与 P/B 同等可信**的标的。

使用 P/E、P/B 两种方法时,隐含股价和隐含市值都要按两种方法分别推导并列展示。不得同时展示两种隐含价,却只用 P/B 计算市值。
- **隐含价**:P/E 隐含价(目标P/E×EPS)、P/B 隐含价(目标P/B×BPS),两行,标清哪个估值方法。
- **市值**:同样两行:市值(P/E估值方法)= P/E隐含价×股本、市值(P/B估值方法)= P/B隐含价×股本。要一个中性值就取两种估值方法均值,但**两个估值方法的市值都要留着让读者看分歧**(交叉核对的意义就在于看两种估值方法差多少)。
- **upside**:vs 现价的隐含涨跌幅也两种估值方法各算一个。

## ★★ 估值倍数对照 sheet(目标倍数的唯一来源 + 放进参照系)
目标倍数必须集中在单独的「估值倍数假设」sheet 中,并说明依据。该表是前瞻目标 P/E、P/B 的唯一来源;「估值」表只引用倍数并计算隐含价。利润率和收入假设集中在「利润与收入假设」中,不得混入估值倍数。

**★ 工作表只允许向前引用(见 `05` §1)**:`历史估值倍数(自身历史区间 + 当前 TTM + 同业 + 相对比值)→ 估值倍数假设(方法说明 + 三层常数 + 三种情景的情绪值/目标 P/B 矩阵)→ 情景切换(读取三种情景的倍数并输出当前情景目标 P/B)→ 业务计算关系 → 利润与收入假设(输出 EPS/BPS)→ 情景估值(当前情景目标 P/B × 前瞻 BPS → 隐含价)→ 估值对比`。
- **估值倍数假设表不得引用后续工作表**。标的自身 forward P/E(现价 ÷ 模型 EPS)需要使用后续计算结果,因此放在「情景估值」的 P/E 交叉核对区;估值倍数假设表只保留同业数据、标的 TTM 和简短说明。
- 逻辑顺序由此变成:**先设置三案倍数(倍数页)→ 切换(情景切换)→ 业务计算关系 → 应用(情景估值)**。

**★ 组织成上下两半(既给证据、又给预测)**:
- **上半【证据】= 市场现在把它当什么定价**:
  - **Forward PE 横向对照表**:将标的与三类对象比较:高确定性成长公司(产业链龙头/平台)、标的所在市场与美股大盘、最直接可比同行,判断市场当前的定价类型。具体对象和数据按标的所在产业链当次获取(例:海力士对照 AI 半导体、S&P/KOSPI 和内存同业)。Forward PE 统一使用下一财年(NTM)共识口径。peer 取当次获取的单一可比值;标的使用 `现价 ÷ 模型下一财年前瞻 EPS` 计算可比值。
  - **加一列 TTM 对照**:周期股盈利暴涨时 TTM 失真(同业 TTM 可数倍于 forward):TTM 列正好展示"盈利暴涨 → forward 压缩"机制,解释标的为何 forward 便宜。
  - 标的 forward PE **逐年**小块(FY26E/FY27E…)+ re-rating 判断 + 自身历史倍数带(看清 TTM 为何虚高)。
  - 这一部分同时用于 **P/E 辅助估值**:列出标的逐年 forward P/E 并与可比公司区间对照(强周期标的仅将 P/E 作为辅助方法,见上「P/B 主要估值方法、P/E 辅助方法」)。

- **下半【预测】= 用三层分解确定目标 P/B(GS 式)**:
  目标 P/B 不直接填入单一倍数,而是分解成三层相乘,每层都有独立来源且可对账(参考 GS/MS 做法):
  ```
  目标 P/B = 历史周期峰值 P/B  ×  结构溢价  ×  情绪值
            (该股【过去】最强周期    (本轮结构性重估幅度,    (周期/情绪位置,
             实际到过的 P/B 峰值)     参考同业相对法)          记分卡驱动)
  ```
  - **第一层 历史周期峰值**:取该股在**过去**最强周期里**实际**到过的 P/B 峰值(MS 等给"历史周期峰值"口径,如 Hynix 2.7x)。★ **绝不能用当下/最近(已被本轮重估抬高)的倍数当基准**:它已含结构溢价 + 情绪,再乘=双重计算。要用"没有本轮 re-rating 时它该值多少"。
  - **第二层 结构溢价**:本轮**结构性(非周期)**的重估幅度。使用**同业相对法**(如 GS:海力士相对三星的历史平均溢价 **30%**→ ×1.30)或可比公司估值区间确定。该参数调整频率较低,但仍需数据支持。
  - **第三层 情绪值**:周期/情绪位置。**1.0 = 市场达到"峰值×溢价";>1.0 = FOMO 情绪溢价(付得比历史峰值×溢价还高);<1.0 = 下行周期折价。** 不确定性最高,由**记分卡**打分映射(见下),不是直接设置。
  - **★ 一致性检验(必做)**:三层相乘**应能复现该股最近的实际倍数**(如 2.7×1.30×1.11 ≈ 3.9x = Hynix 2025末实际 P/B)。复现不了 = 某层口径错,回去校。
  - **目标 P/B = 三层公式算出**(不是蓝字硬填);只有第一、二层常数 + 第三层情绪值是输入。这一行用于「估值」。
  - **非共识假设的表示方式**:高于卖方的目标倍数对应**情绪值 > 1.0**,表示本轮定价强于历史周期。必须同时设置下调条件,见下文情绪持续性。

- **★ 情绪值的持续期**:情绪值 >1.0 时,必须根据历史类比和领先指标估计溢价持续时间,不能默认长期不变:
  - **历史类比(需实时检索)**:查同类资产高情绪定价的持续期。历史上内存周期顶部通常不超过 4 个季度;Cisco 在 dotcom 期间溢价扩张约 3 年,之后 2 年内下跌 88%,25 年后仍未回到原高点。本轮如有售罄和长约提高可见度,持续期可高于常规周期(如 18-24 个月),但仍需设置边界。重资产公司的账面价值可提供部分下行支撑。
  - **情绪值路径**:据此排:可见度内(如 FY26-27)情绪 >1.0,可见度耗尽后(FY28+)跌破 1.0、向"峰值×溢价"之下收敛。
  - **监测领先指标而非股价**:例如下一年产能是否售罄、ASP 是否首次环比下降、竞品是否通过认证、下游 capex 指引是否下调。任一条件出现时,下调情绪值。这些指标同时作为记分卡的调整条件。

- **★ 情绪值只需定性分档**:FOMO 对估值的具体影响难以精确预测。模型的主要前瞻依据是**估值差异**(市场倍数与合理倍数的 GAP%)和**催化剂**(促使差异收敛的事件)。情绪值分为萌芽、启动、过热、退潮等状态,例如启动约 1.0-1.1、过热高于 1.15、退潮低于 1.0,并由「综合判断仪表盘」(`05` 4c)的量价和估值读数支持。
- **★ 量价指标不得进入目标倍数公式**:动量、换手率、乖离和散户净买入可用于判断市场参与度。Baker-Wurgler 情绪指数也包含换手率,且高情绪通常对应较低的后续回报。但若用现价量价拟合目标倍数,再由该倍数计算隐含价,会形成循环论证。量价指标只用于比较市场定价和基本面判断,以及触发情绪状态调整。
- **★ 仪表盘必须通过历史拐点回测**:将「综合判断仪表盘」放回标的上一轮行情拐点,检查当时可获得的基本面、估值差异、催化剂和量价信号能否支持相应判断。不能通过回测的仪表盘只能记录事后结果,不能用于预测。

- **peer 数据实拉**:TTM/forward 倍数、历史峰值 P/B 与结构溢价必须来自可追溯的公开数据、用户提供文件或可选数据工具,统一写入 `input.json.sources`;**绝不用示意数字当数据**。
- **判断标准**:① 目标 P/B 由三层公式计算(修改情绪值时结果会变),不直接填入数值;② 三层相乘能复现最近实际倍数(通过一致性检验);③ 情绪值有记分卡依据,持续性有历史类比;④ 强周期标的 P/E 只作辅助对照(隐含 forward PE),市值由 P/B 推导。

## 必须完成
1. **≥2 估值方法交叉核对**(整体口径)。禁止单估值方法下结论。重资产周期/AI 重估标的:P/B-ROE 和 P/E 两个都做。
2. **逐年隐含价(NVDA 式)**:每年一列 : 目标 P/E × 当年 EPS、目标 P/B × 当年 BPS → 当年隐含股价。
3. **★ 预测表必须回溯过去 + 对照每年代表性股价**:
   - 估值表不能只列未来年份。**同时列出 2021 年起的历史数据**(使用实际盈利和账面数据;缺失时使用有依据的估计值并注明)。
   - 对照价不得全部使用同一个当前价。每年使用该年的代表性股价:① 年末收盘价 ② 年均价(月度均值)。历史年份用真实成交价,通过隐含价与实际价的差异回测模型;未来年份与当前价比较。
   - 这比"模型 vs 单一现价"信息量大得多:看到模型隐含价的逐年轨迹 vs 股票实际怎么走。

## 三情景(Bear/Base/Bull)★★
全局「情景切换」控制明细计算,「估值对比」同时展示三种情景。参数切换和结果展示分开处理。
- **★「情景切换」sheet(放在卖方研报共识之后)是全模型唯一的情景参数表**:① Bear/Base/Bull 下拉框(**默认 Base = 主要情景**)是唯一入口;情景序号由它派生(`=IF(开关="Bear",1,IF(开关="Base",2,3))`:**用嵌套 IF,禁用 MATCH+数组常量 `{"..."}`**,后者在 WPS/部分 locale 下报错,会使相关 INDEX 公式失效);② 每种情景写明触发条件;③ 参数矩阵列出每项可切换参数的 Bear、Base、Bull 数值和一行『当前情景』(`=INDEX(三案行, 案序号)`)。**矩阵布局**:参数名独立一行作为组标题(带底色),下方依次列 Bear、Base、Bull 和当前情景,不要把参数名和 Bear 放在同一行。**说明列分两层**:组标题行说明指标定义、重要性和共同基准;Bear/Base/Bull 各自行说明该情景对应的事件、时点、数值变化和出处。
- **★ 明细计算引用『当前情景』行**:业务驱动量增速、收入强度、价格周期、分部利润率、情绪值等可切换输入均引用「情景切换」的当前情景行。切换后,业务测算、利润、倍数和估值随公式更新。「估值」页改名为「**情景估值**」,输出当前情景的逐年隐含价。
- **★ 参数选择原则**:只切换会显著改变隐含价、且三种情景确实存在差异的少数关键参数(约 5-7 个),其余参数与 Base 共用。可选参数包括业务驱动量增速、收入强度或份额、价格周期、分部利润率和情绪值。参数过多会降低模型可读性。
- **★「估值对比」sheet(三案主输出)**:顶部**三列汇总**(每案 触发条件 / 目标年 NI / 总收入 / 目标倍数 / 隐含价 / vs 现价 / 市值)+ 下方**每案一个逐年 block**(NVDA worked model 式:基准→分部收入→NI→权益→BPS→倍数→逐年隐含价,全公式)。**恒常三列并排、不随开关变**。
  - **★ 引用要求**:对比列**不得引用会随下拉框变化的明细单元格**。各情景列只能引用「情景切换」中对应的 Bear、Base 或 Bull 参数行、三种情景共用的计算参数和静态历史数据。Base 列与明细计算使用相同公式和输入;下拉框为 Base 时,两者结果必须完全一致。
- **下拉框用于切换明细计算**,但「估值对比」必须始终并列展示三种情景,以便在截图或导出后仍能比较价差。
- **盈利必须由各情景的业务参数通过同一组公式计算**,不得直接填入 NI。三种情景共用同一份历史业务数据;修改历史业务数据后,三种情景都应更新。
- **每种情景都要写明完整假设和触发条件**,不能只在 Base 上下增减 20%。参数值必须注明来源,例如卖方共识、管理层口径或最新季报。
- **★ 情景说明的可读性标准**:每个参数先说明定义和重要性,再分别说明 Bear、Base、Bull 对应的事件、时点、数值变化和出处。不得使用"Bear=2027早松提前跌价"一类压缩表达。写作前用研报库语义检索和 web search 核实具体口径;版面不足时增加单元格空间,不要删减必要信息。
- **★ Bear 情景不得低于最新已实现业绩的年化水平**:最新季报已经实现的盈利应计入全年基数。
- **情景矩阵与对比区都要列出 2021-2025 年历史实际值**:① 参数矩阵的历史列填写实际值,三种情景共用相同历史数据;无法计算时标记 n.m.;② 情绪值历史列可按实际 P/B ÷ (峰值×溢价)反推;③「估值对比」各情景区块的历史列使用同一组公式计算,历史隐含价应接近实际年末价,用于检查公式能否复现历史结果。
- 输出三种情景的隐含价及相对现价的涨跌幅。P/B 为主要估值方法的公司只使用 P/B 推导隐含价,P/E 继续按辅助方法规则核对。

## 历史带 overlay(reality check 图)
把 **历史估值带分位 + 现价 + 各情景隐含目标** 叠在一张图(Excel 折线/区间,或 ECharts):
- x = 年;两条带:历史 P/B(或 P/E)的 p25–p75 区间 + min/max 上下沿。
- 叠 3 个点:现价隐含倍数、Base/Bull/Bear 目标倍数。
- **作用**:让读者自己看见"现价是落在历史带内、还是冲到上沿之上(透支)、还是突破带(新 regime)",而不是模型单方面下"市场透支了"的结论。这正是治"模型说 842k vs 现价 2243k"那种分歧的呈现方式。
