# Step 5a — Excel 输出(本仓自包含实现)

> **★ 例 ≠ 规范**:同 `04` 开头约定——本文件是方法论,带具体数字/标的名的例子(标"如/例")来自海力士 worked example,不是规范;新标的现拉现算。
> Excel 的唯一内部实现是 `examples/build_kit.py` + `examples/build_template.py`:前者维护公司无关的结构/格式 helper,后者是新标的起点。公式缓存统一由 `scripts/recalc.py` 跨平台重算。本文件定义二者必须遵守的输出规范。

## build_kit/build_template 的通用规范
建模时遵循以下内部规范:
- **输出契约**:写到 `./out/<ticker>_valuation_model.xlsx`,最终消息回报相对路径,一个模型一个文件。
- **字色**:蓝=硬编码输入 / 黑=公式 / 绿=跨表链接 / 红=外部文件链接。
- **数字格式**:年份文本、`$#,##0`(单位写表头)、三段式零显示 `-`、百分比 `0.0%`、倍数 `0.0x`、负数括号。
- **公式 over 硬编码**:计算格一律公式,假设独立成格用引用;无 #REF/#DIV0。
- **溯源**:**不挂单元格批注(红三角)**——解释/来源一律走**可见注释列**(逻辑列 / 备注列),铁律与落地见 `06-sourcing.md`。
- **重算**:用本仓 `scripts/recalc.py --backend auto <model.xlsx>`;它自动选择 Windows Excel COM、macOS Excel AppleScript 或 LibreOffice headless,直到成功且零公式错误再交付。

> 视觉细化(填色蓝灰板 / 三级边框 / section header merge)以本文件和 `examples/build_kit.py` 为唯一规范来源。

## valuation-model 特有规范

### 0. ★ 第一张 sheet = 封面(报告日期 + 时效性 + Key Takeaways)
研报有时效性(股价/财报/缺口判断都会过期)。**第一张 sheet 必须是封面**,两块:
- **元信息**:标的(名称 + ticker)、**报告日期**、数据截止时点、现价及其时点、关键假设的时效声明(如"基于 2026-06 卖方共识与 capex 指引")。读者一眼知道这份模型是哪天、基于什么时点的数据做的。
- **★ Key Takeaways(核心结论 + 解释最重要的几个指标)**:不要只放元数据。用几条 bullet 讲清这份模型最重要的判断,并**点名解释几个 load-bearing 指标**(它现在多少、意味着什么、为什么重要)——如当下倍数处历史什么位置、核心收入强度/物理锚取多少、周期 normalize 的时点判断、最大的不确定。让读者**看完封面就 get 全局**,再决定要不要进具体 sheet。判断标准:一个没空翻全表的读者,只看封面就能复述"这家现在贵不贵、靠什么、风险在哪"。

### 1. Sheet 结构(标准 13 张, 按标的裁剪)+ ★★ 表序 = 只向前引用的 DAG
`封面 / 历史财务与估值 / 股价走势 / 卖方研报共识 / 历史估值倍数(数据底座: 自身历史带+当下TTM·forward+同业对照+相对核心同行比值) / 估值倍数假设(前置, 三案倍数在此拍) / 情景切换 / 产业链基座(物理锚) / 分部测算 / 利润与收入假设 / 情景估值(当前案) / 估值对比(三案恒常并排, 主输出) / 综合判断仪表盘`
- **历史估值倍数与估值倍数假设拆两页**:先看数据(自己历史上值多少、现在市场给多少、同行值多少、相对核心同行溢价走到哪)再做假设——倍数数据全在前页,假设页只做判断(为什么 P/B + 三层 + 三案矩阵 + 卖方对账)。
- **三情景架构** = 「情景切换」参数库(下拉开关驱动全链, 默认 Base)+「情景估值」(当前案逐年)+「估值对比」(三列汇总 + 每案逐年 block, 恒常可见、防当前案污染)——见 `04`「三情景」。

**★★ sheet 物理顺序必须 = 数据依赖顺序(每张表只引用排在它前面的表,绝不回头引后面的)**。openpyxl 跨表公式虽与物理顺序解耦(引后建的表也能算),但**让"靠后的表引靠前的表"是可读性铁律**——读者顺着 sheet tab 从左往右读,就是顺着因果链读。把估值拆成三张、严格排成 DAG:
- **「利润与收入假设」**:利润率/收入/净利转换/留存**假设** + 由它们驱动的**全部基本面**(分部营收→营业利润→净利→权益→**EPS/BPS**/ROE)。即"P/E 镜头之上的一切"。
- **「估值倍数假设」**:① 上半【证据】forward PE 光谱(标的 forward = 现价[历史财务] ÷ EPS[利润与收入假设],**只引前面**);② 下半【预测】目标 P/E·P/B 逐年。是前瞻倍数的**唯一来源**。
- **「估值」**:只做最后一步——目标倍数(引「估值倍数假设」)× 前瞻每股(引「利润与收入假设」)→ 隐含价 + 市值 + 上行空间。
- **反例(要避免)**:把"标的当下 forward PE"算在倍数表、却去引一张排在它后面的"预测表"的 EPS —— 这是反向引用,读者读到倍数表时那个 EPS 还没出现。修法:把基本面(EPS)前移到倍数表**之前**的表,倍数表只向前引。
> 判断标准:从左到右遍历 sheet,每张表的所有跨表引用目标都已在它**左侧**出现过。出现"引右侧表" = 表序错了,调整拆分/顺序。

### 2. ★ 物理锚基座 sheet 必须打标记
为让 `validate_valuation.py` 能机械识别物理锚(不靠猜 sheet 名),**基座/锚那张 sheet 的 A1 必须包含标记 `[ANCHOR]`**,例如:
```
A1 = "[ANCHOR] 产业链基座 — AIDC CapEx × $/GW(来自 shared-base/compute-aidc-base)"
```
连通性校验靠这个标记定位"链的最底层",所以**不可省略**。

### 3. ★ 逻辑分散进各 sheet,不单开"估值逻辑/故事"页
**不要专门做一张叙述页**(读者要在表与文之间来回跳,乱)。每个关键数的"怎么来的"就近写在**它所在 sheet 的可见逻辑列/备注列**里;方法论一两句放在相关 sheet 顶部或估值表底部即可。Step 1 的锚定备忘录是给 agent 自己用的过程产物,**不落成交付 sheet**。

### 4. ★ 估值默认整体公司、逐年、回溯过去、对照代表性股价
- **整体公司估值,不做 SOTP**(把公司当整体):整体净利 × 目标 P/E → 股价;BPS × 目标 P/B → 股价。SOTP 仅特殊情况。
- **★ 亏损标的:估值 sheet 主镜头用 EV/Sales(不是 P/E·P/B)**——整体或分部亏损 → 该口径用 `营收 × 目标 EV/Sales` 出实数隐含价,P/E·P/B 退诊断行(转正后再上),**绝不让估值 sheet 只剩 NA/0**。目标 EV/Sales 优先 `目标 EV/EBITDA × 稳态 margin` 反推(见 `valuation-lenses.md`)。
- **★ SOTP 也必须逐年**:用 SOTP 时,「情景估值」「估值对比」要**每个前瞻年(2026E-2030E)逐年出各分部 EV → 加总 → 隐含价**,排成逐年阶梯,**禁止只算单一终端年**。下面"逐年排"对 SOTP 同样强制。
- **逐年排(NVDA 式)**:每年一列,目标 P/E×EPS、目标 P/B×BPS → 当年隐含股价。**目标倍数逐年演化(挂 ROE/周期),不是常数**(见 04)。
- **回溯过去**:估值表从 2021 起,历史年份盈利/账面用实际(或往上拍),不要只排未来。
- **对照每年代表性股价**:不要拍单一现价。每年放 ① 年末收盘价 ② 年均价;历史年隐含价 vs 实际价 = 回测。
- **★ 必须有最终市值(双币)**:历史 + 预测两张表都要 `市值 = 股价 × 股本`,并列**原始币(如 KRW)+ 美元(USD)两行**——估值终点是市值,不能只停在每股价。
- **分部占比拆行**:有分部时占比每段单独一行,不压成一句备注。
- **不放 Checks/勾稽页**:校验用外部 `validate_valuation.py` 跑,交付 Excel 保持精简。

### 4b. ★ 可见的重点笔记(让浅读者 get 数字逻辑)
关键 sheet(预测与估值 / AIDC)里,把**重要判断/数字逻辑直接写成可见文字块**(不藏批注),让一个对公司只有皮毛了解的人,看表就能 get 每个数字怎么变、为什么这么变。

### 4d. ★★ 「卖方研报共识」sheet:说人话 + 按"模型假设"组织(服务后面的测算)
共识 sheet 用于记录模型假设的外部参考。读者质疑某个假设时,可在这张表查到市场区间、采用的口径和选择理由。表中条目必须与后续测算一一对应,不得只作研报摘要。
- **直接解释数据**:可见逻辑列使用黑色、句末分行和完整句。不要只堆叠"blended ASP +112%、SOTP HBM 9.2x、P/B 2.16x×1.3 溢价、bit +18.8%"等数字和缩写。每个数据点配一句解释,说明指标定义及其对模型的影响。
- **★ 主体按"模型假设"逐条组织**(不是按 broker 堆):对模型里每个 load-bearing 假设(例: 内存标的为 HBM 增速 / 服务器DRAM / 价格周期 / 供需缺口 / normalize 时点 / 目标 P/B / ROE…),一行写清:**① 街上共识(说人话, 给区间)② 主要分歧 ③ 我们模型取了什么 + 为什么取它**。这样这张表就是后面测算的"卖方对账单"。

推荐结构:① 顶部一句话总览(全街评级 / 目标价区间 / 核心叙事)→ ② **模型假设 × 卖方对照表**(主体, 每个假设一行, 四列:假设 | 街上共识 | 分歧 | 我们取的+为什么)→ ③ 几个最大分歧(决定估值上下限的那 2-3 个)→ ④ 各家一句话立场(broker | 评级/TP | 一句话核心, 不是数字墙)。
- 判断标准:读者拿模型里**任一个假设**,能在这张表查到"街上区间多少、我取了谁口径、为什么";且全程读得懂(没有看不懂的缩写)。

### 4c. ★ 末尾加「综合判断仪表盘」sheet(决策页;含投后跟踪明细)
交付件**最后一张 sheet** = 基于当下信息对公司的**综合判断**(预测引擎),四块 + 一行结论;下方接投后跟踪明细。
- **A 基本面拐点**:业务结构在变好/变坏吗——结构迁移≠周期反弹的证据(mix 迁移、可持续 ROE、盈利可见度),一行定性【强/中/弱】。
- **B 估值错位(预测引擎 ★)**:`市场现在给的倍数(公式算: 现价÷前瞻每股)` vs `基本面该给的倍数(justified)` → **GAP%**。GAP 正且大 = 重估空间(例: 海力士 2025 年中 1.8x vs 该给 3.2x);GAP 转负 = 已透支。这是整张表的预测引擎——错位 + 催化剂能在拐点前看见行情,情绪不能。
- **C 催化剂**:什么会逼市场闭合 GAP,逐条列 `事件 | 时点 | 状态(✓已兑现/进行中/待)`;剩余催化剂决定超涨还能不能撑。
- **D 情绪确认(timing + 刹车,定性档位,不精确量化)**:档位 萌芽/启动/过热/退潮;读数 = 动量/放量、现价倍数 vs justified、散户参与(量价只当温度计,见 `04` 红线);含**衰减扳机**(明年是否还售罄=最强先行、ASP 首次环比跌、竞品认证、capex 下修)。
- **综合判断一行**:A+B+C+D 收成一句可执行的话(该买/该持/该收),并给"上一轮拐点回测"对照行(放回 2025 中这表怎么读)——**验收标准:回测拐点能看到那波,才算装上了预测引擎**(`04`)。
- **下方保留跟踪明细表**:`指标 | 当前值 | 影响的关键假设 | 跟踪方式(数据源/频率)| 触发动作`。

### 5. 分节色带 + 可见逻辑列(美观规范)
- **分节色带(隔断行)**:同一 sheet 内不同 row-group 之间插一行**中蓝底 + 白粗标签**(`305496`),长表分块不糊。
- **可见逻辑列**:假设/测算表右侧留一列「逻辑/来源(整句)」,wrap。**样式**:用**正常黑色**(不要灰、不要斜体——读者反馈灰斜体难读);**句末自动分行(一句一行,不要全挤成一段)**;**不写 markdown 粗体 `**...**`**(Excel 不渲染,会显示成字符)。单元格内分行用 `\n` + wrap_text。
- **文字单元格开头的 "=" 必须转义成全角 "＝"**:说明文字常以公式形开头(如"= capex×强度 + 周期段"),半角 "=" 开头时 Excel 会将其当作公式解析,导致单元格渲染失败。在所有文字写入 helper(lab/logic/txt 等)中统一加入`if t.startswith("="): t="＝"+t[1:]`,不要逐处改写措辞。
- **★ 同组多行共用的说明 → 合并单元格写一次,禁逐行复读**:一组行(如情景矩阵的 Bear/Base/Bull/当前案)的依据相同,就把说明列跨组 merge、整段写一次——不要每行重复"X 档取值(依据见上)"凑数。配套:**`validate_valuation.py` 的溯源覆盖会把合并区的可见逻辑记到它跨到的所有行**(校验器认合并,所以不需要复读)。

### 6. 估计值标注 + YoY + 标签纪律(交付件干净度)
- **估计/派生序列表头标「(估计)」+ 就近一句解释**:如总营收是实际、但 HBM/DRAM/NAND 分部拆分是按占比估的 → section 标「分部营收(估计)」并写明"总额实际、拆分按公司 commentary 估"。
- **营收行配 YoY 行**:每条收入线(总营收 + 各分部)下加一行 YoY(=本年/上年−1),周期股尤其要看增速。
- **★ 交付件标签禁止过程元话语**:band/表头用**干净业务名**。禁止把给 agent 自己看的过程注解写进交付件 —— 如"(可见,不藏批注)""(NVDA式)""(每行只1个)""(粗估)"这类**都不能进最终 Excel 的标签**(粗估口径可放进逻辑列正文,但不进标题)。对应用户铁律「成品不写流程性元话语」。
  - **工程管道话同禁**:"压缩链""参数=『XX』Bear 行""同构自检""引矩阵行""非手拍"这类描述模型内部接线的词,读者不需要也看不懂——band 写业务名(如"Bear 案 — 逐年推演"),接线说明留在代码注释里。实测被用户点名"功能性中间性文字少一些"。
- **★ 缩写/术语在说明文字里首次出现要写全**:OPM→"营业利润率(OPM)"、ASP→"平均售价(ASP)"、BPS→"每股净资产(BPS)"。判断标准同报告:懂金融但不盯这个行业的读者能直接读懂。

### 7. ★ 冻结窗格 + 表头对齐(每张 sheet 必做)
- **冻结窗格(freeze panes)**:每张数据 sheet 都要冻结**顶部标题/表头行 + 最左标签列(A 列)**,长表横竖滚动时锚点始终可见。openpyxl:`ws.freeze_panes = "B3"`(冻结 A 列 + 第 1-2 行);表头占几行就对应改(预测表 note+表头到第 3 行 → `"B4"`;跟踪表表头第 5 行 → `"B6"`)。**不冻结 = 滚到下面就不知道哪列是哪年、哪行是什么。**
- **年份表头右对齐**:数字格右对齐时,**年份/期间表头(2021A/2026E…)也必须右对齐**与下方数字列对齐(默认文本左对齐会和数字错位,很丑)。openpyxl:表头 cell 加 `alignment=Alignment(horizontal="right")`。

### 6. 不做 Sensitivity sheet(已废)
敏感性职能由三情景架构承担(情景切换的杠杆×三案矩阵 + 估值对比三案并排),不再单开 5×5 网格 sheet。理由见 `04-valuation.md`「Sensitivity 网格已废除」。

## 交付硬性要求:公式缓存值必须写回

交付的 xlsx 必须带公式计算结果缓存。运行 `python scripts/recalc.py --backend auto <model.xlsx>` 成功后保存;openpyxl 直接写出的文件公式单元格没有缓存值,下游任何 `data_only=True` 读取都会拿到空值。验收方式:交付前用 `openpyxl.load_workbook(path, data_only=True)` 抽查「估值对比」sheet 的隐含股价行,读得到数字才算交付。
