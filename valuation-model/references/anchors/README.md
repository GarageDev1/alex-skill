# 物理锚菜单(外延机制)

> 这是整个 skill 的"外延"所在。链骨架(②~⑤)和连通性铁律永远不变;**换一类公司 = 换这里的一个物理锚模块**。

## 什么是物理锚
公司需求最底层、可观测、可证伪、能追溯到公开数据的物理量。它是链的最底层输入:
`物理锚 → TAM → 公司营收`。

## ★ 物理锚是两层:共享基座 + 公司切片
```
物理锚 = 【共享基座】(产业链级, 引用别重推 → ../../shared-base/)
       + 【公司切片】(公司级, 本目录各 anchor 文件现做)
```
- **共享基座**:产业链总盘子 + 单位成本拆解(如 AIDC CapEx 总额 + $51.6B/GW 成本树)。**一条产业链建一次,链上所有公司引用同一个**,模型才互相一致。见 `shared-base/README.md`。
- **公司切片**:这家公司吃总盘子的哪一刀 → × 份额 → 营收。这是各 anchor 文件的活儿。
- 建立 compute 公司模型时,第一步先读共享基座,不重新搜索 AIDC 总量,避免与同产业链的其他公司模型使用不同假设。

## 菜单
| 文件 | 公司类型 | 锚 |
|---|---|---|
| `compute-semiconductor.md` | 算力/半导体链 | AIDC 装机 GW × 单位 $ 含量 |
| `consumer-electronics.md` | 消费电子 | 出货量 × ASP × 渗透率/换机周期 |
| `platform.md` | 平台/互联网 | MAU/DAU × ARPU × 时长 |
| `cyclical-commodity.md` | 周期品 | 供需平衡表 → 价格 |
| `consumer-brand.md` | 消费品牌 | 终端动销 + 提价 + 渗透 |
| `saas.md` | SaaS | 席位 × ACV / 用量计费 |
| `float-yield.md` | 浮存金/储备型(稳定币/保险浮存/货基) | 平均 float × 资产端 yield |

## 如何新增一类
1. 在本目录加 `<类型>.md`。
2. 写清:① 锚是什么物理量 ② 怎么从公开数据取 ③ 锚 → TAM → 营收 的 build pattern(公式骨架)④ 一个跑通的范例。
3. **不要碰**链骨架(②~⑤)和连通性铁律 —— 它们公司无关。
4. 在上表加一行。
