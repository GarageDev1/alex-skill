# Step 1 — 接研报 + 选择业务驱动量(默认自动,歧义才停)

> **默认不暂停**:选择业务驱动量决策树命中唯一解时,自动选择业务驱动量 + 选估值方法,把理由写进依据备忘录直接进 Step 2。**仅当出现歧义:公司类型不在菜单里、或 ≥2 个候选择业务驱动量都说得通:才停下让用户确定。** 选错基准 / 选错估值方法,后面全白做,所以自动选了也必须把"为何是它而不是别的"写清楚,供用户事后否决。

## 目标
基于用户提供的研报/材料或公开资料,定下两件事 → 让用户确认:
1. **业务驱动量**(计算起点输入):从 `anchors/` 菜单选一个,或新建。
2. **估值方法**(≥2 个):从 `valuation-lenses.md` 选,做交叉核对。

## 输入
- 用户已有的定性研报、文件路径或粘贴内容;没有时使用公开资料。外部知识库/检索 skill 只可作为可选加速。**直接萃取业务结构,不重复研究、不把无出处的定性材料当硬数字。**
- 研报缺关键维度时补 WebSearch。

## 业务驱动量怎么选(判断流程)
- 公司需求由什么最底层的业务指标驱动?(装机量 / 出货量 / 用户数 / 供需平衡 / 渗透率…)
- 确认该指标**可观测、可证伪,并能追溯到公开数据**。
- 它能不能往下推到公司营收?(基准 → TAM → 份额/单价 → 营收)

## 每个 driver 必须能回答三问(从旧 skill 保留的唯一好东西)
| | 问题 |
|---|---|
| Q1 | 这个 driver 在业务里对应什么业务指标? |
| Q2 | 数据从哪儿来?(具体出处) |
| Q3 | 为什么用它而不是替代的? |

答不上 → 回研报补,**不许带着含糊往下走**。

## 估值方法怎么选
- 看公司类型 → `valuation-lenses.md` 选 ≥2 个。
- **重资产周期 / AI 重估标的**:P/B 和 P/E 之争是真问题,两个都摆,别单边。

## 输出:依据备忘录(会写进 Excel "估值逻辑" sheet)
```
主要计算关系:  [业务驱动量] → [TAM] → [营收] → [利润/账面] → [估值方法] → [隐含价]
业务驱动量: <名称> + Q1/Q2/Q3
估值方法: <估值方法A> + <估值方法B>(为何各自适用 + 怎么三角)
历史数据: 要拉的历史财务 + 历史估值带清单
```

## 选择业务驱动量决策树
```
需求由 AI 数据中心 capex 驱动?            → compute-semiconductor(有现成共享基座)
卖硬件、量×价×换机/渗透驱动?              → consumer-electronics
变现靠用户×单用户(广告/电商/订阅)?       → platform
产品同质、价格由供需缺口决定?             → cyclical-commodity
品牌定价权、动销×提价×渗透?               → consumer-brand
订阅/用量计费、NRR 是核心乘数?            → saas
靠代持一笔钱赚利息/投资收益(稳定币/浮存/货基)? → float-yield(float × yield)
都不像 → 自问"需求最底层那个可观测业务指标是什么",新建一类基准(见 anchors/README)
```

## 从研报萃取哪些维度(选择业务驱动量够用即可,不贪多)
1. **业务/分部结构**:几条收入线、各占比、哪条是价值主要方法。
2. **需求驱动**:每条线的销量/单价由什么业务指标推动 → 候选择业务驱动量。
3. **竞争格局/份额**:公司在主要方法的份额 + 趋势。
4. **盈利特征**:重资产周期 / 稳态 / 高增长未盈利 → 定估值方法。
5. **历史估值习惯**:街历来用什么倍数看它(PB/PE/EV-EBITDA)→ 估值方法候选。
> 只萃取这 5 项的**框架判断**,不抄研报的预测数字(数字 Step 2 才落地)。

## 依据备忘录 → Excel「估值逻辑」sheet(1:1 映射)
备忘录每一项对应「估值逻辑」sheet 的一行,**这张 sheet 是全模型的导读**:
| 备忘录项 | 落到 sheet |
|---|---|
| 主要计算关系 `[基准]→[TAM]→[营收]→[利润/账面]→[估值方法]→[价]` | 顶部一行图示 + 每段指向哪张 sheet |
| 业务驱动量 Q1/Q2/Q3 | 依据义 + 数据出处 + 选它的理由 |
| 估值方法 A/B + 为何各自适用 + 怎么三角 | 估值方法说明段 |
| 历史数据清单 | 指向 历史财务 / 历史估值带 两张 sheet |

## 自动选 + 事后否决(原"卡点"的替代)
- **决策树唯一解** → 不问用户,直接定基准 + 估值方法,依据备忘录(含 Q1/Q2/Q3 + 估值方法选择理由)写进「估值逻辑」sheet 与决策 memo,继续 Step 2 连续完成。
- **歧义路径**(菜单查不到 / ≥2 候选择业务驱动量都成立 / 估值方法组合有真分歧如 P/B vs P/E 之争无法两个都摆) → 一次性展示 ① 主要计算关系 + 候选择业务驱动量各自的 Q1/Q2/Q3 ② 估值方法组合方案,让用户确定后再继续。
- 用户在任何时候否决基准 → 回到本步重选,Step 2-6 重新执行(业务驱动量换了,计算关系中全部数字作废)。
