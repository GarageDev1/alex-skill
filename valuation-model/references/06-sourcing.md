# Step 5b — 溯源核实 + 业务论断

> 每个关键硬编码数字都要有据可查,写进 Excel 的**可见注释列**(逻辑列 / 备注列)。**全簿禁用单元格批注(红三角)**,铁律见下方「全面禁用单元格批注」节。

## ★★★★ 说人话:解释的统一风格(所有逻辑列一律按这个写,最高优先级)
解释不是堆料,是**带读者走一遍推理**。每个驱动的逻辑按这个流程写,别列研报数字清单:
1. **先给假设**(一句话):如"假设 HBM 收入强度(HBM收入÷AIDC capex)在一段时间内大致不变"。
2. **紧跟括号写为什么这假设短期成立**(机制):如"(因为 HBM 是 GPU 物料成本的固定占比、GPU 又是 AI 数据中心 capex 的固定占比,两层占比短期都稳)"。
3. **用实际数据测算/验证**:"按 2025 实际除出来 ≈4.1%,往前几年也除一遍,大致落在 2.9%~4.1%"。
4. **未来路径:只说"为什么这样演化",不重复表格里已有的逐年数字**:"前瞻见上方各年比值——先小降(全球 capex 一时快过它)、再回升(HBM4 放量)、长期缓降(对手认证入局分份额 + capex 基数变大摊薄)"。

**两条硬规矩(治"信息量太大、不知道怎么看"):**
- **不堆卖方具体数字**:**禁止**列"JPM HBM $31B/$46B、bit +38%×ASP 1.7→1.9"这种数据摘录。读者不在乎卖方报了什么具体数,只在乎逻辑。卖方只用**一句佐证**:"数值与 JPM 研报基本相符"或"比 GS 更保守,差异在价端"。完整出处去『卖方研报共识』表查,不在每个格子重列。
- **不重复表格已有的数字**:逐年值表格列里已经有了,逻辑里**别再抄一遍**,只解释"为什么这样走"。
- 判断标准:读起来像**一个人在跟你讲道理**(假设→为什么→一算→为什么这么变),不是像一份数据摘录。机制(为什么)要在,卖方数字清单不要。

## ★★★ 时点判断必须挂物理事件("为什么是这一年,不是别的年")
任何"X 年起缺货/放量/见顶/normalize"的**时点**,**禁止含糊**("2028 起 normalize"是不合格的),必须挂到**具体物理事件 + lead time**:
- **缺货为什么峰值在某年** = 需求已经到了 + 新供给的 lead time 还没到。扩产是物理过程:洁净室建设→设备装机(pull-in)→试产→量产,业界 **2-2.5 年 lead time**——所以下行期(2023-24)没扩的产能,要到 2 年后(2026-27)才补得上,这中间就是缺口峰值。
- **某年起 normalize** = 哪个**具体厂**(如 SK Hynix 龙仁 Y1 / 清州 M15X / Samsung P5)在哪年投产、bit 何时规模放量。要能回答**为什么是 2028 不是 2027**(2027 那些厂只是刚装机/试产,bit 还没规模化)**、为什么不是 2029**(2H28 是多厂放量的交汇点)。
- **同类产能不能一锅算**:新产能去向不同要分清——如 M15X 的新增产能"主要喂 HBM",就**不能算进 conventional DRAM 供给的缓解**(它缓解不了普通 DRAM 缺口)。
- **时点是敏感性变量,不是确定值**:若管理层/部分卖方认为可能拖后(如缺口持续到 2029/2030),要在 Bear/Base/Bull 里体现 normalize 时点的不确定,别把单一年份当铁定。
- **判断标准**:每个"某年起怎样"的论断,都能回答"挂的是哪个物理事件、lead time 多久、为什么偏偏这一年"。挂不上物理事件的纯日历假设 = 不合格。

## 关键驱动数需保留推导过程(build-up)
每个关键驱动(增速/价格/份额/强度…)都要说明数值如何逐步推导(卖方的 build-up 链),不能只写"GS 说 +39%"或填入一个孤立数值。
- 反例(不合格):`server DRAM +39%(GS)` —— 只有结论 + 出处,读者不知道这 39% 怎么来的。
- 正例(合格):`server DRAM bit +39% = AI 服务器出货 +X% × 单台 DDR5 含量 +Y%;AI 服务器随 AIDC capex 放量(GS S/D)` —— 写出量的来源(单元×含量)、怎么挂上 AIDC capex、出处。
- **做法**:读卖方时不要只抄结论数,要扒出他们**怎么搭出来的**(分应用 = 单元×含量;价格 = S/D 缺口的输出;eSSD = AI 机架存储 TB…),把这条 build-up 链写进逻辑列。能挂到物理锚(AIDC capex)的部分**必须显式挂上**(哪怕是间接:量→单元→capex,或价→缺口→AI 吸产能)。
- 判断标准:一个浅了解者读逻辑列,能复述"这个数是从什么推出来的、为什么随 AI 变",而不只是"某家给的"。

### ★★★ 「研报这么写」≠ 论证 —— 必须补研报数字背后的经济机制(为什么)
最高级别要求:**引研报数字时,不能停在"GS 说 +39% 所以我拍 +39%"**——那只回答了"谁说的",没回答"凭什么"。必须额外写出**研报为什么这么写背后的经济逻辑/因果机制**。
- 反例(不合格,仍只是转述):`服务器DRAM +39%(GS, AI服务器出货×单台DDR5含量)` —— 还是在描述 GS 的算法,没讲"为什么 AI 服务器单台含量高、为什么这两个增量相乘出 39%"。
- 正例(合格,讲到机制):`AI 服务器单台 DDR5 含量是通用服务器的 8-12x(一台要喂 8 张 GPU 的数据流, host memory 容量随之倍增), 叠加 AI 服务器台数高增 → bit = 台数增 × 单台含量增 两个增量相乘, 故 +39% 不是线性外推而是乘数效应`。
- 几个该写到第一性原理的典型:
  - **强度法为什么成立**:HBM 收入/AIDC capex ≈ 稳定 ~4%, 因为 HBM 是 GPU BOM 的固定占比、GPU 又是 AIDC capex 的固定占比 → 两层占比相乘结构性稳定(不是巧合)。
  - **内存价为什么能单年 ±100%+**:内存是 commodity, 产能扩张有 1.5-2 年洁净室 lead time → 短期供给刚性, 需求(AI)瞬间爆发 → 缺口的价格弹性极大。
  - **2023 为什么深亏到净利率 -28%**:内存高固定成本(折旧占大头), 价格跌破现金成本仍要生产(停产/重启更贵)→ 价格周期被放大成利润周期, 谷底深亏。
  - **HBM 毛利为什么远高于商品**:准定制 + 认证壁垒(NVDA 认证 1-2 年)+ 供不应求 → 定价权, 不是普通商品的边际成本定价。
- **判断标准**:把所有研报机构名字从逻辑里删掉, 这段逻辑本身**还能自洽地解释这个数为什么成立**。能 → 合格;不能(删了名字就只剩"有人这么估")→ 不合格, 回去补机制。

## ★ 占比/拆分类估计:必须有行业/研报来源 + 写明驱动 why(不能裸标"估")
任何"X 占 Y 的 %"(分部占比、应用结构、份额、渗透率…)**不能只写一个数 + 标"(估)"**,必须:
1. **数字有来源**:引 TrendForce/Counterpoint/GS/Citi 的具体口径(并注明是 **bit 口径还是收入口径**——两者差很大,如 server DRAM 占比 bit 口径 ~50% vs HBM 收入口径溢价后更高)。
2. **写明为什么是这个量级 + 为什么逐年这样变**:如"服务器占传统DRAM 30%→50%,因 ① 云 capex 外溢(GS server DRAM +39%@26)② DDR5 渗透 40%→65% ③ HBM 占比抬升挤压 mobile/PC"。
3. **写出估算换算(从来源数怎么调到这个数)**:不只标来源数,要把"来源口径 → 我用的口径"的换算写出来。如 server% 30→50%:TrendForce 给的是 **server 占总 DRAM 的 bit 口径**(2023=37.6%、2024≈50%),我要的是 **占传统DRAM(剔 HBM)的收入口径** → 剔掉 HBM 那块、再考虑 server RDIMM 单价高于 mobile(收入占比 ≥ bit 占比)→ 估到 ~30%(早年)/50%(2025)。读者能看到"原始数 → 我的数"每一步口径调整。
4. 只写"服务器占传统 DRAM 2025=50%(估)"不足以支持该假设。需要同时说明 50% 的来源、换算过程,以及份额从 30% 上升的原因。

## ★ 分部收入历史拆分:对齐卖方精确模型,别"总量对但 mix 错"
历史分部(HBM/DRAM/NAND/…)优先使用 GS/Citi 等可读研报的逐年 segment 原表,不要在只确认总营收的情况下自行拆分。过往的 2025 年拆分虽然总营收等于 97 兆,NAND 却高估 5.6 兆,传统 DRAM 低估 9 兆,原因是未核对卖方逐年 segment 表。Step 2 取数时应将卖方逐年 DRAM/NAND/HBM 收入原表作为历史基准;2021-22 年卖方不覆盖时,才使用"DRAM~70%/NAND~25%"粗估并标注。

## ★ 措辞:去圈内黑话,写"因为谁/何时/多少 → 所以什么"
可见逻辑/结论里**禁用含糊黑话**:"normalize 回落"、"逐步收敛"、"周期性回归"这类词读者看不懂凭什么。必须落到**具体可证伪的机制**:不是"后随三供份额 normalize 回落",而是"Samsung 2H26 HBM4 量产过 NVDA 认证 → Hynix 份额从 ~70% 滑向 50-60%(让出~15pct)+ 新产能 2H27 放量压 ASP + capex 基数 $59B→$121B 稀释增速"。判断标准:每个"会变化"的论断,都要能回答**谁、何时、多少、为什么**。

## ★ 每个业务线/关键指标必须有"数据论断"(可读性铁律)
**判定标准:一个对公司只有皮毛了解的人,看了也能 get 到三件事。** 每个业务线 + 每个关键指标(产能/份额/ASP/bit/净利率/目标倍数…)都要带:
1. **这个指标在说什么** —— 是什么、什么量纲(如"HBM 后端产能,单位 k wafer/月")。
2. **它的逻辑是什么** —— 怎么推出来的(如"= JPM 业务表 2026 给的 205 kwpm,2028+ 按 capex 指引外推")。
3. **数据来源是什么** —— 哪份研报/披露;**哪怕是粗估,也要写"我基于 X 粗估",不许凭空冒出**。

> 这条直接治旧模型的病:"HBM 后端产能 158→205 这数字怎么来的、为什么扩到 590、来源在哪"全都没说。**关键数字旁边读者必须能立刻回答这三问。**
> 落地:论断写进该行的**可见注释列**。粗估的数字要明写"粗估,基于…"。

## 5 类合法来源(只能是其一)
| 类别 | 例子 |
|---|---|
| 公司披露 | 10-K / earnings call / 投资者日 / IR |
| 行业第三方 | TrendForce / Gartner / IDC / SemiAnalysis / HSBC |
| 卖方共识 | 必须指明哪家 + 哪份 + 日期(JPM 26/1/29…) |
| 历史外推 | "近 5 年 CAGR 推断" / "回归 GDP+spread" |
| 可比类比 | "Intel 2000-2010 路径类比" |

**禁止**:"行业合理水平" / "经验估算" / "卖方共识"(不指明哪家)。

## 落地方式(可见注释列, 行级)
- 每个蓝字输入**行**,在该 sheet 的可见注释列(测算/假设表是「逻辑/来源」列;历史财务等是「备注/来源」列)写一整句:**这个数是什么 / 怎么推出来的(build-up)/ 来源是哪份**(粗估就写"粗估,基于 X")。样式按 `05` §5:正常黑色、非斜体、wrap、句末分行。
- 来源是"从其他公司研究推导"的,也要写清推导链(类比对象 + 路径)。
- helper 用 `examples/build_kit.py` 的 `inp/introw`(写值, **不挂 Comment**)+ `logic`(写注释列);不要自写挂批注的轮子。

## ★★ 全面禁用单元格批注(红三角),一律用可见列做注释
**铁律:任何 sheet 都不挂单元格批注(cell comment / 红三角角标)。** 解释和来源**一律写进一个可见的注释列**(测算/假设表是 N「逻辑/来源」列;历史财务等没有该列的,**在数据右侧加一列「备注/来源」**)。
- **为什么**:批注藏在角标里要悬停才看见,读者打开表根本不知道哪有解释;且批注和可见列重复,造成"三处各写一遍"的信息过载。openpyxl 里**不再调用 `Comment(...)`**,`inp`/`introw` 的 src 一律传 `None`。
- **每个驱动的解释只留一处 = 它所在行的注释列整句**;来源简引写进正文("与 JPM 基本相符"/"GS 2/8"),完整出处(URL/页码)统一去『卖方研报共识』表。
- **不另开"读法/方法"叙述段**(会和注释列重复);方法论一两句放 sheet 顶部即可。

## ★★★ 注释列写"为什么",不是复述表格已经显示的现象(解释 ≠ 描述)
最容易犯的懒:逐年值表格里已经画出来了(先降、再升、后降),注释里又用文字把这个走势复述一遍——**等于没解释**。读者看表就知道"先降",他要的是**为什么会降**。
- 反例(只描述,不合格):"2026 先小降(全球 capex 扩得比 HBM 快)、2027 回升(HBM4 放量)、2028 后缓降(份额被切+基数变大)" —— "capex 扩得比 HBM 快"只是把"比值下降"换个说法,没讲清**为什么** Hynix 的 HBM 收入这一年追不上 capex。
- 正例(讲机制,合格):"2026 略降——这年钱大量砸在厂房/电力/网络/GPU 整机上,HBM 只占一小块,且 Hynix 的 HBM 产能还在 ramp、基本卖光扩不出更多,所以它的 HBM 收入一时跟不上 capex 的爆发;2027 龙仁新产能投产、Hynix 供给跟上,比值修复"。
- **判断标准**:把表格里那串数字遮住,注释文字本身能不能告诉读者"为什么是这个走势"。能 → 解释;只是把数字走势翻译成中文 → 还是描述,回去补 why。

## 完整性校验(行级)
