# 验收清单 — Valuation Model

> 交付前逐条过。任一未过 → 回对应 Step 修正再交付。

## 链路完整性(核心)
- [ ] **连通性**:改 ① 物理锚的一个底层输入,隐含股价跟着动(没动 = 链断 = 不合格)
- [ ] **公式引用完整**:没有未被任何下游公式引用的 sheet 或 driver 行
- [ ] **不截断**:估值用的利润/账面来自 Step 3 链路,不是硬塞的卖方数字
- [ ] **④ 段完整**:营收 → 毛利 → 净利 → 账面/FCF 这一段存在(不是营收直接甩给估值)

## 历史底座
- [ ] **基年真实**:③④ 投影起点 = 历史财务真实值(无 plug/反算)
- [ ] **倍数在带内**:目标倍数落在历史估值带内;突破带有明确论证
- [ ] **现实检验**:隐含价阶梯叠在历史股价带上,现价位置被解释(分位/regime)

## 估值
- [ ] **多镜头**:≥2 估值镜头,有三角验证(重资产周期标的 P/B + P/E 都做)
- [ ] **三情景**:Bear/Base/Bull,每档有 source + 触发条件;情景切换 + 估值对比承担区间分析职能
- [ ] **估值对比**:三案恒常并排,每案沿同一条链推到隐含价,且含隐含 forward P/E 交叉验证

## 溯源
- [ ] 每个关键硬编码输入行有可见注释列(5 类来源之一,不许"行业合理水平")
- [ ] 全簿无单元格批注/红三角;解释和来源不藏在 hover 里
- [ ] 可见注释列讲清三层:数据 / 逻辑 / 情景差异

## Excel 工程
- [ ] Font/Fill/Border/三段式 format 全部合规(blues+greys+white)
- [ ] 用 `python scripts/recalc.py --backend auto <model.xlsx>` 重算成功,并确认零 #REF!/#DIV/0! 等错误
- [ ] 公式 over 硬编码(投影/margin/PV/三情景链路 都是 Excel 公式)
