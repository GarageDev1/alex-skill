# 产业链共享物理基座(shared-base)

> **这是「物理锚」的第一层——产业链级、公司无关、建一次反复用。**
> 第二层(公司切片)在 `../references/anchors/<类型>.md` 里现做。

## 为什么要有这一层

物理锚不是一个东西,是**两层**:

```
物理锚 = 【共享基座】(本目录, 引用别重推)  +  【公司切片】(anchors/, 每家现做)

第一层 共享基座 (产业链级):
   ① 产业链总需求预测 (如 全球 AIDC CapEx $/年, 2022A–2036E, 带源)
   ② 单位成本拆解 → 各部件占总盘子的 %

第二层 公司切片 (公司级):
   ③ 这家公司吃哪一刀 (NVDA=GPU+网络 / Hynix=HBM+DRAM+NAND / TSMC=wafer)
   ④ 切片 TAM = 产业链总盘子 × 部件占比%  → × 公司份额 → 公司营收
```

**已发现的问题**:Hynix 旧模型没有引用共享基座,而是另行引用 HSBC 的 HBM TAM,与 NVDA 模型使用的 $830B AIDC 预测不一致。共享基座要求同一产业链的公司模型引用同一个总量假设。

## 白送的护栏:产业链一致性

所有公司从同一个基座切片后,得到一条单公司模型给不了的 sanity check:

- **同一个 spend tree 节点,被几家公司同时声称时,各家的隐含 $/GW 之和 ≤ 该节点的 $/GW。**
- ⚠ 注意**产业链 resale 叠加**:Hynix 卖 HBM 给 NVDA,HBM 的钱已经包在 NVDA 的 GPU 收入里。所以**公司营收会跨层叠加、可以 > 某个 tree 节点**;护栏只对「最终需求口径的 spend tree(MECE)」成立,不能拿各家营收直接相加去对总额。见 `compute-aidc-base.md` 的口径说明。

## 文件约定

每条产业链一组文件:
- `<chain>-base.json` — 机读 SOT(模型直接 import)
- `<chain>-base.md` — 人读版(同样数据 + 推导逻辑 + 出处叙述)

当前基座:
| 文件 | 产业链 | 总盘子 | 单位拆解 |
|---|---|---|---|
| `compute-aidc-base` | AI 算力/数据中心 | 全球 AIDC CapEx | $51.6B/GW 成本树 |

## 维护规则

- **更新频率**:每季财报季后修一次(hyperscaler capex 指引会变),改 `.json` 并 bump `meta.version` + `meta.updated`。
- **新产业链**:加一组 `<chain>-base.{json,md}`,在上表加一行;骨架(②~⑤)和连通性铁律一个字不动。
- **建模时**:Step 1 选到对应锚 → 读这里的基座当 Layer 1 → 只做公司切片;建模把基座落成 Excel 里一张「产业链基座」sheet(带源),公司切片 sheet 用公式引用它,保持连通 + 自包含可审计。
