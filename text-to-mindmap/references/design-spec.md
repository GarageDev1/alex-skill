# 设计系统与主题说明

改模板、加主题、排查观感问题时看这里。

## 布局:竖向逻辑树(唯一形态)

从上到下:

1. **Hero**:主题带(kicker + 大标题 + 导语 + 标签),给全图定调。
2. **主干**:一条贯穿左侧的竖线(`.tree::before`),所有分类挂在上面。
3. **一级分支(分类)**:主干上一个**彩色圆环节点** + 一小段**横线** → 彩色**分类标题**(带下划线)。
4. **二级分支(要点)**:分类下垂一条**浅色子干**,每条要点用**横线 + 小圆点**连到**内容块**。

即用户要的"线—线—点—块"结构:主干线 → 分支横线 → 圆点节点 → 内容块,层层连起来。

## 配色纪律(为什么不花)

- **一个分类只用一种颜色**。分类之间换色,分类内部所有连线、节点、表头、等级条、图表点都用这一种色。避免"一屏七种颜色互相打架"。
- 正文文字一律中性灰黑,层级靠**字号、字重、留白**拉开,不靠彩色文字。
- 表格程度值(高/中/低)用**同色系等级条**(1/2/3 格),不用红橙绿——红橙绿三色一上来就乱。
- 重点行用**分类色的极浅底**(6% 混合)点出,不用重描边。

这套纪律是从三轮返工里收敛出来的:第一版七色卡片 + 红橙绿标签 + 彩色术语 → 被判"颜色排版混乱";改成单色系后才立住。详见 `worked-example.md`。

## 三套美术主题(同一布局,换皮)

| 主题 | 底色 | 标题字体 | hero | 适合 |
|---|---|---|---|---|
| `business-blue` | 白底白卡 | 无衬线 | 深蓝渐变 | 通用、商务、投研 |
| `magazine-light` | 米色亮底 | 衬线(宋/思源宋) | 米色平铺 | 人文、行业观察、杂志感分享 |
| `tech-dark` | 全暗底暗卡 | 无衬线 | 近黑渐变 + 霓虹 | 科技、产品发布、年度总结 |

主题只改这些 CSS 变量(见 `assets/template.html` 顶部):中性色(`--ink/--line/--trunk/--page/--card`)、hero 一组、`--meter-empty`、`--title-font`。**分类主色不在主题里定**,由 `render.py` 的 `PALETTES[theme]` 按分类顺序分配(暗色主题用更亮的霓虹色),JSON 给了 `color` 则覆盖。

## 加一套新主题
1. 在 `template.html` 加 `html[data-theme="你的名字"]{ ... }`,覆盖上面那组变量。
2. 在 `render.py` 的 `THEMES` 元组和 `PALETTES` 字典各加一条(7 个分类色)。
3. 用示例 JSON 渲一张回看验收。

## 尺寸与渲染
- 内容固定宽 1180 CSS px,`--force-device-scale-factor=2` 出 2360px 高清。
- Chrome 无头截图给的是超高窗口(8000),再由 PIL 按背景色扫描裁掉底部空白;`--width` 时 LANCZOS 精确缩放。
- 象限图坐标系:viewBox 380×250,原点 (52,222),x 轴到 360、y 轴向上到 22;JSON 的 x/y(0–1)线性映射到这个范围。

## 常见观感问题
- **中文变方框**:运行机缺中文字体,装 PingFang / 思源黑体 / Noto Sans CJK。
- **某条要点被切**:文字太长,拆短或改 `table`/`units`。
- **表格右侧溢出**:列太多或某列文字过长,精简列或缩短文案。
- **暗色主题分类色发灰不亮**:用了 JSON 里为浅色配的 `color`,去掉让它走暗色调色板。
