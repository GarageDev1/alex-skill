# 估值章接法(估值数字唯一真相源 = 估值模型)

本文件管第四章「盈利预测与估值」的**数字怎么来**:报告只"讲"不"算",所有量化数字由 `valuation-model` skill 的"物理锚 → 营收 → 利润 → 多镜头 → 隐含价"通电链产出,报告从模型读出。写作与呈现规范沿用 ICA:[`assets/研报写作规范.md`](../../initial-coverage-advanced/assets/研报写作规范.md)(写作)+ [`references/阶段5-报告组装.md`](../../initial-coverage-advanced/references/阶段5-报告组装.md)(vs 卖方共识对标表 / GAP% / 催化剂 / 风险提示的报告呈现);Obsidian 格式见 [`obsidian-格式覆盖.md`](./obsidian-格式覆盖.md)。本文件只管 VM 数字接法。

## 核心原则(三条,贯穿全章)

1. **估值主权在模型**:目标价、评级、三情景的唯一来源是模型的隐含价阶梯;一个 ticker 只许有一套估值数字,报告不另建加权目标价。
2. **因果方向铁律**:**先把 Excel 模型建完、validate 通过、跑出隐含价阶梯,报告结论(目标价/评级)再从模型读出来**——绝不允许先在脑里定结论、再搭 Excel 去凑。模型是报告的前置事实源,不是事后插图。
3. **模型的信息量必须进报告**:物理锚链(锚→TAM→营收→利润→镜头→隐含价)、三情景假设、隐含价 vs 历史估值带、可跟踪仪表盘指标——这些是第四章正文素材,不是 Excel 里的暗数据。报告中所有可量化判断(分部占比、margin、目标倍数)必须与 `input.json` 一致;写报告发现对不上,回模型裁决,不在报告里另拍一个数。

## 〇、第四章小节骨架(连续编号,唯一切分 —— 防断号根治)

第四章「盈利预测与估值」的小节**必须按下表连续编号 4.1→4.N,不留任何无编号的裸 `###` 小节**。本表是第四章小节切分的**唯一真相源**,**覆盖 ICA 报告模板那套「4.1盈利 / 4.2估值 / 4.3风险」的 3 节切分**——因为本 skill 把估值内容拆成了下方更多并列模块(物理锚链 / 多镜头 / GAP% / vs共识 / 历史band / 催化剂),它们各自是独立小节、都要有 4.x 编号。

| 小节 | 内容 | 对应本文 § |
|---|---|---|
| 4.1 物理锚链 | 锚→营收传导(Q1/Q2/Q3) | §五 |
| 4.2 盈利预测 | 分部收入 / 利润逐年预测 | §三表② |
| 4.3 估值分析 | 多镜头三角 + 三情景汇总 + 隐含价阶梯 + 足球场图 | §六 + §三表①③ |
| 4.4 自身历史 P/E·P/B band | 当前 / 目标倍数在自身历史区间何处 | §九 |
| 4.5 市场预期与估值争议 | 估值错位 GAP% + vs 卖方共识对标表 | §八 + §七 |
| 4.6 催化剂与跟踪指标 | 未来催化剂 + 各争议追踪指标速查 | §十一 |
| 4.7 风险提示 | 8-12 条,分类 | — |

**铁律(防的就是历史断号 bug)**:

- 上表每个模块都**必须有 4.x 编号**。历史 bug 根源:GAP%(§八) / vs共识(§七)在 ICA 的 3 节骨架里**没有编号槽**,生成时被裸挂成无编号 `### 估值错位 GAP%` / `### vs 卖方共识`,卡在 4.3 和 4.4 之间 → 4.3→4.4 断号。本骨架给每个模块都分了 4.x,杜绝裸挂。
- **编号必须连续无断号**,小节顺序可按公司微调,但不许跳号、不许裸标题。
- 4.5 的 GAP% 与 vs共识 可**合为一节**(下用加粗子段分两块)或**拆成 4.5 / 4.6 各一节**(后续顺延 4.7 催化剂 / 4.8 风险),两种都行,但都**必须带 4.x 编号**。
- 标题带判断(沿用写作规范),如 `### 4.5 市场预期与估值争议:现价已超基本面该给的`。

## 一、软依赖:没模型就先触发跑

第四章开始前,同目录 `{ticker}_valuation_model.xlsx` 必须存在。**不存在 → 按名调用 `valuation-model` skill**,跑其全部 6 步(管线模式:Step 1 自动选锚;Step 6 只出 memo,无旧报告可对账)。本 skill 的研究稿(第一章)即其"定性研报"输入。

## 二、模型交付门(进第四章前必须通过,FAIL 阻断)

valuation-model 6 步跑完后,**必须**以两步收尾,缺一不得进第四章:

1. **跑 `recalc.py`**:`python "<VM_SKILL_ROOT>/scripts/recalc.py" --backend auto "<model.xlsx>"`(自动选择 Windows Excel COM、macOS Excel AppleScript 或 LibreOffice headless 回写缓存值,第四步抽表/取数全靠它)。
2. **跑 `validate_valuation.py`**:`python "<VM_SKILL_ROOT>/scripts/validate_valuation.py" "<model.xlsx>"`,**verdict=FAIL 必须回修再交付**——尤其「$B 口径统一」「结构(物理锚通电)」「连通性」三项 fail = 模型不合规,严禁带病进报告。

历史财务与预测全部由 valuation-model 的 Step 2/3 承担(历史底座 + 物理锚链)。本 skill 不自建 DCF/三表。

## 三、模型报表强制嵌入(数字同源铁律)

报告必须把模型的逐年明细以 Markdown 表嵌进正文,**不是只放三情景单点汇总**(分部收入、各项预测、历史财务与估值整页都要整页呈现,不出简略版)。

**统一抽取**:`python "<VM_SKILL_ROOT>/scripts/extract_model_tables.py" --full "<model.xlsx>"` 一次抽出**四张表**,按下方分配嵌入。**前置**:抽取前模型必须已跑 `recalc.py`。

| 表 | 抽取参数 | 嵌入位置 | 内容 |
|---|---|---|---|
| ① 三情景估值汇总 | `scenario_table` | 估值分析 | Bear/Base/Bull 的净利/总收入/目标倍数/隐含价/隐含 forward P/E/vs 现价/隐含市值 |
| ② 盈利预测表 | `--forecast` | 盈利预测 | Base 逐年(分部收入+YoY+占比/总营收/段 OPM/营业利润率/净利/净利率/EPS/BPS/ROE,2021A-2030E) |
| ③ 三情景隐含价阶梯 | `--ladder` | 估值分析 | 三案逐年隐含价(2026E-2030E × Bear/Base/Bull),让读者看"哪一年长进现价" |
| ④ 历史财务与估值整页 | `--with-history` | 历史财务分析 | 分部收入/占比/毛利率/净利/权益/复权股本/FX/年末·年均股价/复权 EPS·BPS/实际 P/E·P/B/市值,含 TTM + 最新季报列 |

**铁律**:每张表的每个数字**只能由脚本抽取、绝不手敲**——手敲必然与模型漂移。报告正文叙述的任何收入/利润/EPS/目标价/情景数字,必须与这些嵌入表逐字一致。单位以模型原生为准($B 财务 / $ 每股 / 本币股价 / x 倍数),表题标清口径,不在导出环节做币种换算。

## 四、三张"判断" sheet 必读(模型判断力最集中处)

用 XLSX skill 读模型时,**除数字表外必须额外读这三张判断 sheet**——报告估值的"观点"全从这里来:

1. **「综合判断仪表盘」**:估值错位 GAP%(市场倍数 vs justified 倍数)+ 基本面拐点 + 情绪档位/衰减扳机。→ 喂 §八 GAP% + §十一催化剂。
2. **「历史估值倍数」**:自身历史 P/E·P/B 带 + 同业 + 相对核心同行比值。→ 喂 §九 历史 band。
3. **「卖方研报共识」**:每个模型假设"街上多少 / 我取了什么 / 为什么"。→ 喂 §七 vs 卖方共识。

## 五、物理锚链叙述(本章核心,必须写透)

用业务语言讲清"锚 → 营收"的传导——锚是什么物理量、为什么是它(Q1/Q2/Q3 三个问题答清)、共享基座给出的总盘子、公司切到哪一刀。配传导图(桑基/散点,见 [`obsidian-echarts适配.md`](./obsidian-echarts适配.md) 模型语义图)。

**叙述里的具体驱动量与"为什么这个数",必须从模型的「物理锚基座 / 分部测算」逐年驱动量 + 各 sheet 可见逻辑列读出,不从研究稿定性重写**——否则报告与模型在判断层脱节(两张皮)。三情景的物理锚假设差异(Bear/Base/Bull 各自假设锚怎么走)也要讲清。

判断标准:非专业读者读完能复述"什么物理量驱动这家公司值多少钱"。

## 六、多镜头三角验证

≥2 镜头(PE/PB/DCF/SOTP 等,按公司选)各自的隐含价 + 为何适用此公司。**权重与隐含价以模型「综合判断仪表盘」为准,不在报告里另定固定加权**。讲清"哪个镜头主导、为什么",不机械平均。配估值足球场图(横条,各镜头区间 + markLine 当前价 + 目标价)。

## 七、vs 卖方共识(从「卖方研报共识」sheet 逐假设读出,不另写)

把该 sheet 里每个 load-bearing 假设的"街上多少 / 我取了什么 / 为什么"落成**逐行对标表**:

| 指标 | 本研究 | 卖方一致预期 | 差异及分歧根源 |
|---|---|---|---|
| 下一财年收入增速 | {%} | {%} | {谁对量产/份额更激进} |
| 毛利率 | {%} | {%} | {分歧根源} |
| EPS | {值} | {值} | {分歧根源} |
| 盈利转正/拐点时点 | {年份} | {年份} | {分歧根源} |
| 目标价 | {值} | {值} | {分歧根源} |

差异大处必须在"分歧根源"列**点名解释**(谁对量产节奏/份额/毛利假设更激进)。卖方覆盖稀少时如实说明"覆盖少、仅 N 家",用现有 1-2 家也进表对比。**这是 alpha 叙述,不是跟随**——只在散文里提一句共识倍数不算数,必须摊成表。

## 八、估值错位 GAP%(强制,进「市场预期与估值争议」)

从「综合判断仪表盘」读出"市场现在给的倍数(现价÷前瞻每股)vs justified 倍数 → GAP%",讲清重估空间:**GAP 正且大 = 重估空间 / 转负 = 已透支**。这是模型的预测引擎,必须进报告,不能只谈共识倍数就过。

## 九、自身历史 P/E·P/B band(从「历史估值倍数」sheet 读,不现拉)

标注当前价 / 目标价倍数落在自身历史区间何处 + 相对核心同行比值。配一张历史 forward P/E(及 P/B)band 图。让读者判断"当前估值在自身历史贵不贵"。

## 十、P/E·P/B 历史 vs 预测口径铁律(防错)

盈利预测/估值摘要表里,**历史年(A)列的 P/E·P/B 必须用当年实际股价÷当年 EPS/BPS**(实际倍数,与「历史财务整页表」的"实际 P/E·P/B"逐字一致);**只有预测年(E)列**才用现价÷当年预测值(forward 倍数)。

**绝不允许用现价去除历史年的 EPS/BPS、再把结果当作该历史年的 P/E**——那是把今天的价套到过去的年份上(如把现价÷2025EPS=151x 当成"2025A 的 P/E",实际 2025A 用 2025 年末价算只有 83x)。行标题分别标清"历史=当年价÷值 / 预测=现价÷值",历史与预测交界处倍数跳变属正常(价格基准从历史价换成现价),必要时一句话说明。

## 十一、催化剂与跟踪指标速览(从仪表盘 + 各争议追踪指标读)

引用决策 memo 的证伪触发器 + 「综合判断仪表盘」的基本面拐点 / 情绪档位 / 衰减扳机,写明"什么读数出现 = 回模型改数重算"。配两张表:

- **未来催化剂**:`时点 | 事件 | 预期影响(超预期 +% / 不及预期 -%)`
- **各争议追踪指标速查**:`对应争议 | 追踪指标 | 可证伪阈值 | 当前值`
